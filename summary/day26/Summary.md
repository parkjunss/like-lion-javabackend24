## 1. 수업 시작 및 복습 (JavaScript 전반)
- **도입:** 강의의 흐름을 잡기 위해 지난 시간에 배운 내용을 복습하는 것이 중요하며, 특히 **비동기 처리**와 관련된 내용에 대한 복습이 필요함.
- **전체 복습의 중요성:** 배울 내용이 너무 방대하고 복잡하게 느껴질 수 있으나, 차근차근 학습하고 필요할 때 개념을 떠올리는 습관이 중요함.

## 2. JavaScript 실행 모델과 비동기 처리의 필요성
- **단일 스레드 구조:** 브라우저에서 동작하는 자바스크립트는 한 번에 하나의 작업만 처리하는 **단일 스레드(Single Thread)** 모델을 사용합니다.
- **브라우저의 역할:** 자바스크립트가 동작할 때, HTML 파싱, CSS 렌더링 등 다른 작업들은 사실 자바스크립트가 아니라 **브라우저 자체(Browser API)**가 담당합니다.
- **블로킹 문제:** 만약 자바스크립트 코드가 시간이 오래 걸리는 무거운 작업을 처리하게 되면, 브라우저가 멈춘 것처럼 보이며(Blocking), 사용자는 불편함을 겪게 됩니다.
- **비동기 처리의 필요성:** 따라서 자바스크립트가 오래 걸릴 것 같은 작업들은 메인 스레드를 막지 않도록 **비동기(Asynchronous)** 방식으로 처리하는 것이 필수적입니다.
- **결과처리 구조:** 비동기 처리가 완료된 후 해야 할 일들은 자바스크립트가 메모리에 가지고 있을 수 없기 때문에, 반드시 **콜백 함수(Callback Function)**에 담아 전달해야 합니다.

## 3. 비동기 처리의 발전 과정 (콜백 $\rightarrow$ Promise $\rightarrow$ Async/Await)
- **비동기 처리의 발전:** 비동기 처리를 구현하는 과정은 콜백 함수를 이용하는 것에서 시작되었으나, 순서가 복잡해지고 오류 처리도 어려워지는 문제가 있었습니다.
- **Promise 객체:** 이 문제를 해결하기 위해 **Promise** 객체가 도입되었습니다.
    - Promise는 비동기 작업의 최종 성공 또는 실패를 나타내는 객체로, `.then()` (성공 시 처리)와 `.catch()` (실패 시 처리)를 이용해 안정적인 로직 흐름을 구축할 수 있게 했습니다.
- **Async/Await:** Promise 체인도 여러 번의 비동기 작업이 연결되면 복잡합니다. 이를 더욱 간결하게 처리하기 위해 **`async`와 `await`**가 도입되었습니다.
    - `await`를 사용하면 비동기 작업이 완료될 때까지 기다리며 코드를 작성할 수 있어, 비동기 코드를 마치 동기 코드처럼 읽기 쉽게 작성할 수 있게 합니다. (내부적으로는 콜백과 Promise의 장점을 활용하여 자동으로 처리해줍니다.)

## 4. JavaScript의 핵심 데이터 구조와 기능
- **배열과 객체:** 자바스크립트는 배열과 객체 형태로 데이터를 구조화하고 처리할 수 있습니다.
- **자료구조 조작:**
    - `push()`, `pop()`: 배열의 마지막 요소에 데이터를 추가하거나 제거합니다.
    - `shift()`, `unshift()`: 배열의 첫 번째 요소를 제거하거나 추가합니다.
    - `map()`, `filter()`, `forEach()`: 배열의 데이터를 반복적으로 처리하며 변환하거나 필터링하는 **고차 함수(Higher-Order Function)**가 핵심입니다.
- **함수(Function)의 위상:** 자바스크립트에서는 함수 자체가 하나의 **일급 객체(First-Class Object)**로 취급되어, 변수에 할당하거나 다른 함수의 인자로 전달하는 것이 매우 일반적입니다.
- **함수형 프로그래밍:** 자바스크립트는 함수형 프로그래밍 관점에서 접근하는 것이 효과적이며, 이 부분이 중요하게 다루어져야 합니다.

## 5. 자바스크립트 객체 모델과 최신 개발 트렌드
- **객체와 자바의 차이점:**
    - **자바:** 클래스 정의가 엄격하며, 객체를 생성하기 위해 `new` 키워드를 사용해야 합니다.
    - **자바스크립트:** 클래스 정의 없이도 간단한 객체로 인스턴스화될 수 있습니다. 객체 생성 후에도 필드나 메서드를 추가할 수 있어 유연성이 높습니다.
- **프로토타입(Prototype):** 자바스크립트는 객체들이 동일한 기능(메서드)을 공유할 수 있도록 **프로토타입**을 사용합니다. 이는 메모리 효율성을 높이는 핵심 메커니즘입니다.
- **ES6와 클래스 문법:**
    - ES6(ECMAScript 2015)에서 도입된 **`class` 문법**은 문법적으로는 클래스처럼 보이게 했지만, 내부 동작은 여전히 **함수(Function)**와 **프로토타입**을 기반으로 작동합니다.
    - 따라서 자바스크립트를 이해할 때는 '클래스 문법'보다는 **'함수 기반의 프로그래밍 방식'**으로 이해하는 것이 중요합니다.
- **외부 라이브러리의 발전:**
    - 과거에는 브라우저마다 다른 환경을 수동으로 처리해야 했기 때문에 jQuery와 같은 라이브러리가 큰 인기를 끌었습니다.
    - 하지만 현재는 React와 같은 최신 프레임워크들이 복잡한 환경 문제를 해결해 주면서, 개발자들은 순수한 자바스크립트(Vanilla JS)와 같은 효율적인 방식으로 개발을 지향하고 있습니다.

## 1. 객체 구조 분해 및 스프레드 연산자 활용
- **객체 구조 분해 (Destructuring):**
    - 객체를 통해 값을 얻을 때, 구조 분해를 이용하면 객체에 포함된 값을 하나하나 꺼내지 않고 한 번에 깔끔하게 추출할 수 있습니다.
    - 예시: `Person` 객체에서 `name` 필드값만 즉시 가져와 사용할 수 있습니다.
- **배열 구조 분해:**
    - 객체뿐만 아니라 배열에서도 구조 분해를 사용하여 값을 사용할 수 있습니다.
    - **차이점 (필드 vs. 순서):**
        - **객체:** 필드 이름(속성)이 중요하며, 순서와 상관없이 필요한 속성만 꺼낼 수 있습니다.
        - **배열:** 순서가 중요합니다. 배열은 인덱스(순서)를 통해 값을 가져와야 합니다.
- **스프레드 연산자 (`...`):**
    - 객체나 배열의 속성/요소들을 복사하거나 펼쳐서(Spread) 사용할 때 유용합니다.

## 2. 자바스크립트의 DOM 요소 접근 방법
- **DOM 선택자 이용:**
    - `document.querySelector()`를 사용하여 CSS 선택자(Selector)를 이용해 원하는 HTML 요소를 가져올 수 있습니다. (예: `#btn`과 같은 ID 선택자 사용)
    - **다른 요소 선택 방법:** `getElementById`, `getElementsByClassName`, `getElementsByTagName` 등의 전통적인 방식도 존재합니다.
- **코드 실행 위치의 중요성 (타이밍):**
    - **핵심 주의점:** 자바스크립트는 위에서 아래로 순차적으로 실행됩니다. 스크립트가 실행될 때, 아직 HTML 요소가 화면에 그려지지 않은 상태라면 해당 요소를 찾을 수 없어 `null`을 반환하게 됩니다.
    - **권장 방식:** 스크립트는 가능한 한 **HTML 본문(`<body>`) 끝 부분**에 작성하는 것이 좋습니다. 그래야 브라우저가 모든 HTML 요소를 먼저 그린 후 자바스크립트가 동작하며 요소들을 읽어올 수 있습니다.
    - **기존 방식 (History):** 과거에는 `onload` 같은 이벤트로 전체 화면 로딩을 기다리게 했지만, 현재는 자바스크립트가 DOM 요소를 잘 인지하고 있어 단순한 위치 배치가 더 중요합니다.

## 3. 이벤트 처리와 리스너 구현
- **이벤트의 정의:**
    - '이벤트'란 버튼 클릭, 글자 입력, 마우스 움직임(호버) 등 **특정 사건(특정 일이 발생했을 때)**을 의미합니다.
- **이벤트 리스너 (Event Listener):**
    - 특정 HTML 요소에 '귀를 달아준다'는 개념입니다. 즉, 해당 요소가 어떤 특정 이벤트(클릭, 마우스 오버 등)가 발생했는지 *항상 듣고 있다가* 이벤트가 발생하면 미리 정의된 함수(콜백 함수)를 실행하도록 설정하는 과정입니다.
- **구현 문법 (Standard):**
    - `요소.addEventListener(이벤트이름, 이벤트핸들러[함수])` 형태로 사용합니다.
    - **매개변수:**
        1. **이벤트 이름 (Event Type):** `click`, `mouseover`, `keydown` 등 발생시킬 사건의 이름을 지정합니다.
        2. **이벤트 핸들러 (Event Handler):** 이벤트가 발생했을 때 실제로 실행될 함수를 지정합니다. (이 함수를 정의하고 전달합니다.)
        3. **캡처링 여부 (Option, 선택):** 이벤트가 전파되는 방식을 제어합니다. (기본적으로는 생략 가능하며, 나중에 심화 학습합니다.)

## 4. 이벤트 전파와 이벤트 핸들러 활용
- **이벤트 전파 (Event Bubbling):**
    - HTML 요소들이 박스 모델처럼 계층적으로 쌓여있기 때문에, 이벤트가 발생하면 그 이벤트가 **최초 발생 지점에서 시작하여 부모 요소들을 거쳐 바깥으로(위로) 전파**되는 현상을 말합니다.
    - 예시: 가장 안쪽의 `<li>`를 클릭하면 그 부모인 `<ul>`과 `<div>` 등도 영향을 받습니다.
- **이벤트 종류의 다양성:**
    - `click` 외에도 `dblclick` (더블클릭), `mouseover` (마우스 진입), `keydown` (키보드 입력), `keyup` (키보드 해제) 등 매우 다양한 종류가 존재합니다.
- **함수의 일급 객체 취급:**
    - 자바스크립트에서 함수는 일반 값처럼 취급되기 때문에, 이벤트 리스너의 인자로 직접 전달하거나 별도의 함수로 정의하여 재사용할 수 있습니다.

## 5. 실습: 이벤트 기반 카운터 구현
- **DOM 값 조작의 방법:**
    - 특정 요소의 값을 변경할 때는 `innerHTML` (HTML 태그를 포함할 경우) 또는 `textContent` (순수한 텍스트 값만 다룰 경우) 속성을 사용합니다.
- **구현 원리:**
    - 버튼 요소에 `addEventListener`를 사용하여 각각의 클릭 이벤트를 걸어줍니다.
    - **핵심 로직:** 이벤트가 발생할 때마다 카운터 변수(상태)를 업데이트하고, 이 새로운 값을 DOM 요소에 다시 출력(값 재할당)하여 화면에 변화를 줍니다.
    - 이를 통해 버튼 클릭이라는 '이벤트'와 '상태 관리'를 결합하여 동적인 기능을 구현할 수 있습니다.

## 1. HTML 요소를 효율적으로 찾아내는 방법 (Selector 활용)
- **선택자 사용의 중요성:** 웹 요소(Element)를 JavaScript에서 찾아내어 값을 얻거나 상호작용하게 만드는 것이 프로그래밍의 핵심입니다.
- **요소 식별의 우선순위:**
    - **ID 사용:** 가장 간편하고 확실하게 요소를 찾는 방법입니다. (단, ID는 페이지 내에 고유해야 함)
    - **클래스(Class) 및 태그(Tag) 사용:** ID를 사용할 수 없는 경우 `document.getElementsByClassName()`이나 `document.getElementsByTagName()` 등을 활용할 수 있습니다.
    - **CSS 선택자 활용 (권장):** `document.querySelector()`와 `document.querySelectorAll()` 사용이 가장 범용적이고 강력합니다.
- **선택자의 반환값 이해 (핵심):**
    - **`querySelector()`:** 선택자에 맞는 요소를 **가장 먼저 하나만** 반환합니다.
    - **`querySelectorAll()`:** 선택자에 맞는 모든 요소들을 **NodeList 형태의 리스트**로 반환합니다.
- **리스트 순회:** 여러 개의 요소를 가져왔을 경우, 리스트의 길이(`length`)만큼 반복문(Loop)을 돌면서 각각의 요소에 개별적으로 이벤트를 걸어주어야 합니다.

## 2. 이벤트 리스너를 이용한 웹 동적 상호 작용 구현
- **이벤트 리스너 등록:** 특정 요소에 이벤트(예: 클릭 이벤트)를 감지하는 `addEventListener()` 메서드를 사용합니다.
- **이벤트 발생과 기능 구현:**
    - 버튼 클릭과 같은 이벤트가 발생했을 때, 원하는 동작(예: 배경색 변경, 값 계산)을 수행하도록 함수를 연결합니다.
    - **동적 화면 변경:** JavaScript의 핵심 목표는 사용자 상호작용에 따라 화면을 **동적으로** 변경하는 것입니다.
- **값의 읽기/쓰기:**
    - **데이터 읽기:** `element.innerHTML` 또는 `element.innerText`를 사용하여 요소의 내부 텍스트나 HTML 값을 추출할 수 있습니다.
    - **상태 변경:** 이벤트 발생 시, 현재 요소의 값을 읽어와서(예: 초기값) 증가(++) 또는 감소(--)시키는 방식으로 상태를 관리하고 화면에 다시 반영할 수 있습니다.
- **다중 요소 처리:** 여러 개의 동일한 버튼이나 요소를 리스트로 가져온 후, 반복문을 통해 각 요소마다 개별적인 이벤트 리스너를 추가해야 합니다.

## 3. 이벤트 객체(Event Object) 이해 및 활용 (e.target)
- **이벤트 발생 시 전달되는 객체:** 이벤트 리스너의 콜백 함수는 첫 번째 인자로 **이벤트 객체(Event Object)**를 받습니다.
- **이벤트 객체의 활용:**
    - **`e.target` (가장 중요):** 이벤트 객체가 갖고 있는 `target` 속성은 **실제로 이벤트를 발생시킨 원본 요소**를 정확하게 반환합니다.
    - 이 속성을 이용하면 버튼의 `value`를 읽어오거나, 특정 요소를 식별하는 등 이벤트를 발생시킨 시점의 상태를 파악할 수 있습니다.
- **함수 정의 방식과 `this`의 관계:**
    - 일반 함수로 정의한 경우, 이벤트 핸들러 내에서 `this`가 이벤트 발생 요소와 연결될 때가 많아 직관적입니다.
    - 화살표 함수(Arrow Function)를 사용하면 `this`의 컨텍스트가 바뀌는 경향이 있으나, 이벤트 객체 자체를 통해 `e.target`을 사용하면 문제없이 해당 요소를 얻어낼 수 있습니다.

## 4. 이벤트 리스너 제거 및 고급 응용
- **이벤트 리스너 제거의 필요성:** 이벤트가 한 번만 발생하고 더 이상 동작할 필요가 없을 때, 메모리 누수 방지 및 기능 정지 등을 위해 리스너를 제거해야 합니다.
- **제거 방법:** `removeEventListener()` 메서드를 사용합니다.
- **제거 시 주의사항 (매우 중요):**
    - `removeEventListener`를 사용할 때는 **반드시 이벤트를 등록할 때 사용했던 동일한 함수(Function Reference)**를 인자로 전달해야 합니다.
    - 따라서 이벤트 핸들러를 외부 변수나 함수로 명확하게 정의해 두고 사용해야 제거가 가능합니다. (내부에 익명 화살표 함수로 정의하면 제거가 어렵습니다.)
- **최상위 레벨 이벤트 등록 (Document Body):** `document`나 `body` 전체에 이벤트를 걸어놓으면 (위임 방식), 특정 자식 요소의 클릭 여부를 `e.target`을 통해 판단할 수 있습니다.

## 1. 이벤트(Event)의 이해와 기본 구현
- **이벤트 정의:** 특정 사건(사건 발생)이 발생했을 때 이를 감지하고 처리하는 프로그래밍 개념입니다. 이는 웹 브라우저뿐만 아니라 일반 애플리케이션 전반에 걸쳐 사용됩니다.
- **이벤트 처리의 핵심 원리:**
    - 이벤트를 발생시키기 위해서는 이벤트가 발생할 **대상(Target)**이 중요합니다. (예: 버튼, 이미지 등)
    - JavaScript를 사용하여 특정 HTML 요소를 찾아내야 이벤트를 걸 수 있습니다.
- **HTML 요소 선택 방법 (JS 활용):**
    - `getElementById()`: 아이디를 기준으로 요소를 찾습니다.
    - `querySelector()`: CSS 선택자를 이용해 요소를 찾을 수 있습니다.
- **이벤트 리스너 등록 (`addEventListener`):**
    - `addEventListener`는 특정 요소(대상)가 어떤 이벤트를 기다리도록 "귀를 달아주는" 역할을 합니다.
    - **주요 매개변수:**
        1. 이벤트 이름 (예: 'click', 'submit')
        2. 이벤트 핸들러 (이벤트가 발생했을 때 실행될 함수)
        3. 캡처링 여부 (Boolean):
            - 기본값은 `false`입니다. (버블링 방식)
            - `true`로 설정하면 캡처링 방식으로 이벤트가 전파됩니다.
- **이벤트 객체와 `this` 키워드:**
    - 이벤트 핸들러 함수 내부에서는 첫 번째 인자로 **이벤트 객체(Event Object)**를 받습니다. 이 객체를 통해 이벤트가 발생한 요소의 정보를 얻을 수 있습니다.
    - **`this`의 범위:** 일반 함수로 정의할 경우 `this`가 이벤트가 발생한 객체를 가리키지만, 화살표 함수로 사용하면 `this`의 범위가 달라지므로, 명시적으로 이벤트 객체에서 타겟을 가져오는 것이 안전합니다.
- **이벤트 제거:**
    - 이벤트 리스너는 `removeEventListener()` 함수를 사용하여 제거할 수 있습니다.
    - **주의사항:** 이벤트를 제거할 때는 등록할 때 사용했던 **이벤트 타입, 대상 요소, 핸들러 함수**를 정확하게 일치시켜서 넣어주어야 합니다.

## 2. 이벤트 전파(Event Propagation)의 원리 및 제어
- **이벤트 전파란?**
    - 하나의 요소(예: 빨간 박스)를 클릭하는 이벤트가 발생했을 때, 해당 요소뿐만 아니라 그 바깥에 위치한 모든 부모 요소(핑크 박스, 바디, HTML 태그 등)까지 순차적으로 이벤트가 전달되는 현상을 의미합니다.
- **세 가지 전파 방식:**
    - **버블링(Bubbling):** (기본값) 이벤트가 가장 안쪽 요소에서 발생하여 바깥쪽으로 거품처럼 퍼져나가는 방식입니다. (→ 기본 동작)
    - **캡처링(Capturing):** 이벤트가 가장 바깥쪽(문서 객체)에서 시작하여 안쪽으로 내려오는 방식입니다. (→ `capture: true` 설정 시)
- **이벤트 전파 제어 방법:**
    - **`e.stopPropagation()`:** 이벤트가 전파되는 과정을 강제로 중단시키는 메소드입니다. 특정 요소에서 이벤트 처리를 완료하고 더 이상 상위 요소로 이벤트가 올라가는 것을 막을 때 사용합니다.

## 3. 폼(Form) 태그를 이용한 이벤트 핸들링 실습
- **`<form>` 태그의 기본 이벤트:**
    - `<form>` 태그는 기본적인 **`submit`** 이벤트를 가지고 있습니다.
    - 사용자가 폼 내의 버튼을 클릭하여 폼을 제출(Submit)하려고 할 때, 이 기본 이벤트가 먼저 작동합니다.
- **기본 동작 방지 (Prevent Default):**
    - 기본적으로 폼이 제출되면 정의된 `action` 속성으로 페이지가 이동(새 페이지 로딩)합니다.
    - 자바스크립트에서 이 기본 동작을 막기 위해 **`e.preventDefault()`**를 반드시 호출해야 합니다.
- **데이터 값 가져오기:**
    - 폼 내의 각 `<input>` 요소들은 `name` 속성을 기반으로 JavaScript에서 값을 가져와 처리할 수 있습니다.
- **복합적인 이벤트 구현 예시 (회원가입 폼):**
    - **이벤트 등록:** 폼 요소 자체에 `submit` 이벤트를 리스너로 등록합니다.
    - **유효성 검사 (Validation):**
        1. **필수 값 체크:** ID나 비밀번호 등 필수 입력 필드가 비어 있는지(`null` 또는 빈 문자열) 확인합니다.
        2. **값 비교:** 비밀번호와 비밀번호 확인 값이 일치하는지 비교하여 검증합니다.
    - **처리 흐름:**
        - 유효성 검사를 통해 모든 값이 올바른 경우에만 데이터를 서버로 전송합니다 (실제로는 AJAX 등을 사용).
        - 유효성 검사에 문제가 있을 경우, 사용자에게 경고 메시지를 표시하고 커서를 해당 입력 필드로 이동(Focus)시켜 안내합니다.

## 1. 현대 웹 애플리케이션의 데이터 요청 방식 비교
- **전통적인 폼 제출 방식 (Action):**
    - 폼의 `submit` 버튼을 누르면 서버로 전체 페이지를 요청(Full Page Request)합니다.
    - 서버는 폼의 인풋 상자(Input)에서 `name` 속성과 `value` 쌍을 이용해 데이터를 추출합니다.
- **SPA 기반의 비동기 데이터 요청 방식 (AJAX/Fetch):**
    - 페이지 전체를 요청하는 것이 아니라, 필요한 데이터만 비동기(Asynchronously)로 가져옵니다.
    - 데이터 전송 시 폼 요소의 `name`이 없을 수 있으며, 이때는 JavaScript의 `id`를 사용하여 값을 읽어 데이터를 전송하는 것이 일반적입니다.

## 2. 싱글 페이지 애플리케이션(SPA)과 성능 최적화 필요성
- **싱글 페이지 애플리케이션 (SPA)의 정의:**
    - 사용자가 어떤 상호작용을 하더라도 URL이 바뀌지 않고, 화면의 내용만 동적으로 변경되는 웹 구조입니다 (예: SNS 댓글 창).
    - 전체 페이지를 새로고침하는 대신, JavaScript가 서버로부터 필요한 데이터만 받아와서 화면을 조작(조작)합니다.
- **DOM 조작의 성능 문제:**
    - SPA 환경에서 내용이 빈번하게 바뀌면 브라우저가 DOM을 수시로 재조작(Re-rendering)해야 하며, 이 과정에서 성능 저하(비효율성)가 발생합니다.
- **가상 DOM(Virtual DOM)의 활용:**
    - **개념:** React와 Vue 같은 라이브러리는 실제 DOM 대신 JavaScript 메모리 상에 가상 DOM(Virtual DOM)이라는 DOM과 동일한 구조를 만듭니다.
    - **작동 원리:** 실제 DOM에 직접 조작하는 대신, 가상 DOM에서 변경 사항을 빠르게 처리한 후, 실제 DOM과 가상 DOM을 비교(Diffing)하여 *실제 변경된 부분만* 브라우저에 그려냅니다. 이를 통해 성능을 극대화합니다.

## 3. DOM 조작 및 이벤트 처리 기본기
- **DOM 요소 접근 및 값 추출:**
    - `document.getElementById()` 또는 `querySelector()`를 사용하여 원하는 요소를 가져옵니다.
    - **값 추출 방법:**
        - 인풋 상자의 실제 입력 값: `.value` 속성을 사용합니다.
        - 태그 내부의 텍스트 값: `innerText`를 사용합니다.
        - HTML 구조 전체: `innerHTML`을 사용합니다.
- **폼 이벤트 기본 제어:**
    - **이벤트 기본 동작 방지:** `e.preventDefault()`를 사용하면 폼이 자체적으로 수행하는 기본 제출 동작(페이지 이동/새로고침)을 막을 수 있습니다.
    - **이벤트 핸들러 구성:** 이벤트 리스너는 함수를 별도로 정의하거나 화살표 함수(`=>`) 형태로 간단하게 처리할 수 있습니다.
    - **폼 전송 제어:** 로그인 성공 등 특정 로직이 완료된 후 페이지 이동이 필요할 경우, `return` 등을 사용하여 폼 제출 이벤트가 정상적으로 진행되도록 유도해야 합니다.

## 4. 동적 콘텐츠를 위한 고급 이벤트 처리: 이벤트 위임
- **문제 인식 (Before Event Delegation):**
    - To-Do 리스트처럼 목록 항목(L.I.)이 동적으로 생성될 경우, 새로 생성된 모든 `L.I`마다 개별적으로 이벤트 리스너를 등록해야 합니다. 이는 코드가 복잡하고 관리하기 어렵습니다.
- **이벤트 전파(Event Bubbling)와 이벤트 위임(Event Delegation) 원리:**
    - **원리:** 이벤트가 발생한 요소(예: `L.I`)에서 시작하여 DOM 트리를 따라 상위 요소(예: `<ul>`)로 전파되는 특성을 이용합니다.
    - **적용 방법:** 이벤트 리스너를 동적으로 생성되는 개별 요소가 아닌, **부모 요소 (`<ul>`)**에 한 번만 등록합니다.
    - **핵심 로직:** 이벤트가 발생하면, 이벤트 핸들러 내부에서 `event.target`을 사용하여 *실제로 이벤트를 발생시킨 요소*가 무엇인지(즉, `L.I`인지) 확인하고, 그 요소의 내용물이나 속성을 읽어와 필요한 로직을 구현합니다.
- **장점:** 새로운 `L.I`가 추가되어도 부모 `<ul>`에 걸린 하나의 이벤트 리스너가 모든 이벤트를 처리할 수 있으므로, 코드의 효율성과 확장성이 매우 높아집니다.

## 1. 자바스크립트와 SPA(Single Page Application)의 기본 원리
- **JS를 이용한 화면 제어:** 자바스크립트를 활용하여 DOM(Document Object Model)을 조작함으로써 화면을 동적으로 변경할 수 있다.
- **SPA의 동작 원리:** 싱글 페이지 애플리케이션(SPA)은 서버에 페이지 전체를 요청하는 것이 아니라, 필요한 데이터만 요청한다. 자바스크립트가 이 데이터를 받아와 HTML 태그를 조작(교체, 추가 등)하여 화면을 구성한다.
- **DOM API 활용:**
    - `getElementById()`와 같은 DOM API를 이용하면 가져온 HTML 태그 자체가 하나의 객체로 사용된다.
    - 이 객체로부터 속성을 가져오거나, 함수를 이용해 엘리먼트를 생성하고 조립하는 데 활용할 수 있다.

## 2. 자바스크립트 단축 평가 (Short-circuit Evaluation)
- **개념:** 논리 연산자(`&&`, `||`)가 내부적으로 평가되는 원리를 활용하여 코드를 간결하게 만드는 방식이다.
- **`&&` 연산자:** 첫 번째 피연산자가 `false` (Falsy)일 경우, 뒤의 코드는 실행되지 않고 해당 값을 반환한다. (A가 참일 때만 B가 실행됨)
- **`||` 연산자:** 첫 번째 피연산자가 `true` (Truthy)일 경우, 뒤의 코드는 실행되지 않고 해당 값을 반환한다. (A가 거짓일 때만 B가 실행됨)
- **활용 예시:** 조건문(if문)을 사용하지 않고, 값이 비어 있거나 특정 조건을 만족할 때만 코드를 실행하도록 제어할 수 있다.

## 3. ES6+ 고급 문법 및 코드 간결화 기법
- **요소 판단 기준 (Falsy 값):** 자바스크립트가 `false`로 판단하는 값들은 다음과 같다.
    - `null`, `undefined` (값이 존재하지 않거나 명시적으로 비어 있는 값)
    - `0` (숫자 0)
    - `''` (빈 문자열)
- **옵셔널 체이닝 (Optional Chaining, `?.`):**
    - 객체 속성 접근 시 해당 속성이나 객체가 `null` 또는 `undefined`일 경우, 오류(Error)가 발생하는 것을 방지한다.
    - 예: `person?.job?.manager?.name`과 같이 사용하면, 중간 단계의 속성이 없어도 프로그램이 오류 없이 안전하게 동작한다.
- **널 병합 연산자 (Nullish Coalescing, `??`):**
    - 값이 `null` 또는 `undefined`인 경우에만 기본값(Default Value)을 할당할 수 있게 해준다.
    - 기존의 삼항 연산자 (`A == null ? B : A`)를 훨씬 간결하게 대체할 수 있다.
- **코드 간결화의 의의:**
    - 이들 문법(`?.`, `??`, 단축 평가)은 자바스크립트 문법이 발전하며 개발자들이 코드를 더 짧고, 안전하며, 간편하게 작성할 수 있도록 추가된 기능들이다.

## 4. 학습 및 코드 작성 시 주의사항
- **코드 가독성과 간결성:** 현대 개발 경향은 짧고 간결한 코드를 선호하는 경향이 있지만, 코드를 읽는 다른 개발자들도 충분히 이해할 수 있도록 가독성이 높은 코드가 중요하다.
- **학습 방법:** 처음부터 모든 문법을 외우려 하기보다, 원리를 충분히 이해할 수 있도록 코드를 풀어 써보는 과정이 중요하다. 이해가 되면 자연스럽게 간결한 코드를 사용하고 싶다는 생각이 들게 된다.
- **실전 이해:** 이러한 단축 평가와 새로운 문법들을 알고 있는 것은 코드를 읽을 때 '암호'처럼 보이는 코드를 해석하는 데 큰 도움이 된다.

4. 단축 평가 : 논리 연산자 이용해 조건에 따라 값을 반환하거나 코드를 실행하는 방식

a && b는 a가 참처럼 취급되는 값이면 b를 반환하고
a가 거짓처럼 취급되는 값이면 a를 반환한다.
특정 값이 존재할 때만 뒤의 코드를 실행하고 싶을 때 주로 쓴다.

a || b는 a가 참처럼 취급되는 값이면 a를 반환하고,
a가 거짓처럼 취급되는 값이면 b를 반환한다.
주로 기본값을 설정할 때 사용한다.

null, undefined, 0, ""은 자바스크립트에서 거짓으로 취급 그래서 0, "" 도 값으로 인정하는 ?? 사용이 안전
