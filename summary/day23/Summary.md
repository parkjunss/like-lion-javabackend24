## 1. 학습 태도 및 마음가짐 다지기
*   **인사 및 관계의 중요성:** 기술적인 능력 외에도 기본적인 사회적 상호작용(예: 인사)이 직장 생활 등 여러 상황에서 중요하다.
*   **지속적 노력의 어려움:** 꾸준히 무언가를 유지하는 것이 어렵기 때문에, 마음의 느슨함을 이해하고 꾸준한 각오를 다지는 것이 필요하다.
*   **일상에서의 반성 습관:** 잠자리에 들기 전 '오늘 하루를 뿌듯하게 보냈는지', '아쉬웠는지' 등을 스스로 생각해보는 습관을 들이는 것이 마음가짐과 자세를 다지는 데 큰 도움이 된다.

## 2. 웹 동작 원리 이해: 서버와 클라이언트 구조
*   **웹의 작동 방식:** 웹은 단순한 파일 실행이 아니라, 클라이언트(사용자의 브라우저)와 정보를 제공하는 서버가 **상호 통신**하며 동작한다.
*   **HTTP 프로토콜의 역할:** 클라이언트가 서버에 페이지 요청(Request)을 보내면, 서버가 해당 정보를 찾아 응답(Response)하는 과정 전체가 HTTP라는 프로토콜을 통해 이루어진다.
*   **웹 서버 vs. 웹 애플리케이션 서버:**
    *   **웹 서버 (Web Server):** 정적인 자원(Static Resource, 예: 이미지, 일반 HTML 파일)을 요청에 따라 찾아서 그 내용을 그대로 응답하는 역할을 한다.
    *   **웹 애플리케이션 서버 (WAS):** 요청이 들어왔을 때, 내부 프로그램(Java, Python 등)을 실제로 동작시켜 처리한 후, 그 결과를 바탕으로 **동적(Dynamic)인 페이지**를 만들어 응답한다.
*   **중요한 개념:** HTML 태그의 문법을 모두 알기보다, **'웹이 어떻게 동작하는지'** 서버-클라이언트 간의 정보 주고받기(데이터 흐름) 원리를 이해하는 것이 더 중요하다.

## 3. HTML 구조 이해: 박스 모델과 시맨틱 마크업
*   **HTML과 XML:** HTML은 XML(eXtensible Markup Language)이 확장된 형태이다. 모든 웹 문서는 구조와 규칙을 가지고 있는 '독특한 문서 유형(DOCTYPE)'을 정의해야 한다.
*   **문서의 기본 구조 (XML 관점):**
    *   모든 XML 기반 문서는 반드시 **루트 태그**로 감싸져야 한다 (시작 태그부터 끝 태그까지).
    *   **엘리먼트 (Element):** 시작 태그부터 종료 태그까지를 의미한다.
    *   **속성 (Attribute):** 태그에 추가 정보를 부여하는 것.
    *   **Head와 Body:**
        *   **`<head>` 영역:** 브라우저가 읽는 메타정보(Meta information)를 포함한다.
        *   **`<body>` 영역:** 실제 사용자에게 보여줘야 하는 내용만 포함한다.
*   **박스 모델 (Box Model):** 모든 HTML 요소는 보이지 않더라도 기본적으로 **'박스'** 형태로 존재한다.
    *   **블록 레벨 (Block Level):** 현재 요소가 가지고 있는 내용의 크기와 상관없이, **한 줄 전체를 차지**하며 배치된다. (예: `<div>`, `<p>`)
    *   **인라인 레벨 (Inline Level):** 내용물이 차지하는 영역만큼만 크기가 결정되며, 옆으로 배치될 수 있다. (예: `<span>`, `<img>`)
    *   **인라인 블록 (Inline-Block):** 내용물처럼 공간을 점유하면서도(인라인), 크기 지정 및 독립적인 배치가 가능하도록 만든 개념이다.
*   **시맨틱 태그 (Semantic Tag):**
    *   `<div>`를 사용하는 대신 `<header>`, `<main>`, `<section>`, `<footer>` 등 의미를 부여하는 태그를 사용한다.
    *   이들은 단순히 박스 역할을 넘어, **관련 있는 내용들끼리 하나의 묶음(바구니)**으로 정리하여 웹 페이지의 구조와 관리 편의성을 높여준다.

## 4. CSS 스타일링 원칙: 배치와 속성 제어
*   **CSS의 역할:** HTML은 '무엇을 보여줄지(내용 뼈대)'를 담당하고, CSS는 '어떻게 보여줄지(스타일)'를 담당하여 역할을 명확히 분리하는 것이 좋다.
*   **CSS 적용 방식 (권장 순서):**
    1.  **외부 스타일 시트 (External Style Sheet):** 별도의 `.css` 파일을 만들고 `<link>` 태그로 연결하는 방식 (가장 권장됨).
    2.  **내부 스타일 시트 (Internal Style Sheet):** HTML 파일의 `<head>` 태그 안에 `<style>` 태그로 작성하는 방식.
    3.  **인라인 스타일 (Inline Style):** 태그 자체에 `style` 속성을 직접 부여하는 방식 (전체적으로 수정이 어려워 권장하지 않음).
*   **핵심 개념:** 웹 페이지를 구성하는 모든 요소는 HTML이라는 박스 안에, 다시 헤더, 본문 등의 구조적인 박스들이 들어있는 **'박스 안에 박스'** 형태로 관리된다는 것을 항상 기억해야 한다.

## 5. CSS 선택자 마스터하기: 요소 식별의 핵심
*   **선택자의 중요성:** CSS와 JavaScript는 HTML 태그를 꾸미거나 조작하는 것이 목적이므로, **"어떤 HTML 태그에 스타일을 적용할 것인지"**를 정확하게 지정하는 '선택자(Selector)'가 가장 중요하다.
*   **선택자의 종류와 규칙:**
    *   **1. 태그 선택자 (Tag Selector):** 태그 이름을 직접 지정한다. (예: `p`, `div`)
    *   **2. 클래스 선택자 (Class Selector):** `.`(점)을 붙여 사용한다.
        *   **특징:** 같은 스타일을 가지는 여러 요소에 **재사용**할 때 사용한다. (가장 많이 사용됨)
        *   **강점:** 하나의 태그에 여러 개의 클래스를 부여할 수 있어 여러 스타일을 조합할 수 있다.
    *   **3. ID 선택자 (ID Selector):** `#`(샵)을 붙여 사용한다.
        *   **특징:** 페이지 내에서 **유일한(Unique)** 고유의 요소 하나를 지정할 때 사용한다.
        *   **주요 활용:** 주로 JavaScript에서 특정 요소에 값을 넣거나 조작할 때 많이 활용된다.
*   **활용의 초점:** 복잡한 페이지를 분석할 때, 눈으로 보이는 모든 것을 '큰 박스 $\rightarrow$ 작은 박스 $\rightarrow$ 내용물'로 쪼개서 바라보며, 어느 부분이 어떤 *선택자*로 관리되고 있는지 이해하려는 노력이 중요하다.

## 1. HTML 폼 태그 이해
- **폼 태그의 역할:** 사용자가 웹 페이지를 통해 값을 입력하고 서버에 정보를 전달할 때 사용하는 태그입니다.
- **이벤트 기반:** `<form>` 태그는 기본적으로 이벤트를 가지고 있습니다. 특히 `submit` 버튼을 누르면 설정된 `action`으로 이동하는 방식으로 약속되어 있습니다.
- **값 전달 원리:**
    - `<input>` 태그와 같은 입력 요소는 `name` 속성을 통해 값을 서버에 전달합니다.
    - 서버는 이 `name` 속성(변수명)을 이용해 사용자가 입력한 실제 값(`value`)을 전달받아 처리합니다.

## 2. CSS 박스 모델 이해 (Padding 및 Margin)
- **패딩 (Padding):** 요소의 **콘텐츠**와 **테두리(Border)** 사이의 내부 여백입니다.
    - 패딩 값을 주면 박스의 크기가 커지는 것이 눈에 띄게 관찰됩니다.
    - 패딩은 기본적으로 [위(Top), 오른쪽(Right), 아래(Bottom), 왼쪽(Left)] 순서로 적용됩니다.
    - 패딩 값을 줄이는 방법은 1개, 2개, 3개, 4개의 값을 지정할 수 있으며, 4개일 경우 순서대로 적용됩니다.
- **마진 (Margin):** 요소와 **다른 요소** 사이의 외부 간격입니다.
    - 마진은 다른 태그와 태그 사이의 간격을 확보하는 역할을 합니다.
    - **마진 상쇄 (Margin Collapsing):** 위에 있는 요소의 하단 마진과 아래에 있는 요소의 상단 마진이 단순히 합쳐지는 것이 아니라, 두 값 중 더 큰 값 하나만 적용되어 간격이 줄어드는 현상입니다.

## 3. 고급 CSS 속성 및 기본값 초기화
- **브라우저 초기화 (Reset):**
    - 브라우저마다 HTML 태그에 기본적으로 부여하는 패딩이나 마진 값이 다를 수 있습니다.
    - 이러한 스타일 차이로 인해 일관성 없는 디자인이 생기는 것을 방지하기 위해, CSS 코드를 작성할 때 초기화(Reset) 코드를 사용하여 기본값을 제거하거나 통일하는 것이 좋습니다.
- **표시 속성 활용:**
    - **`visibility: hidden`:** 요소 자체가 눈에 보이지 않게 숨겨지지만, **공간(Space)은 차지**합니다.
    - **`display: none`:** 해당 요소를 아예 화면에 표시하지 않아, **공간도 차지하지 않게** 만듭니다.
    - **`opacity` (투명도):** 요소의 투명도를 조절하는 속성입니다. 값은 0(완전 투명)부터 1(불투명)까지 변화하며, 크기나 위치를 변화시키지 않습니다.
- **배경 설정:**
    - `background-image`를 사용하여 상자에 배경 이미지를 적용할 수 있습니다.
    - 이미지가 반복되는 것을 막고 싶을 때는 `background-repeat: no-repeat` 속성을 사용합니다.
- **테두리 (Border):**
    - `border` 속성 하나로 테두리의 굵기, 스타일(실선, 점선 등), 색상 등을 한 번에 정의할 수 있습니다.
    - 테두리는 컨텐츠 크기에 더해지므로, 실제 박스 크기가 달라지는 것을 고려해야 합니다.

## 4. CSS 레이아웃 구성 방법 (Float, Position)
- **플로트 (Float):**
    - 특정 요소를 마치 물 위에 띄우듯 (floating) 주변 콘텐츠 옆에 배치할 때 사용합니다.
    - 주로 메뉴나 목록처럼 여러 요소를 옆으로 나열할 때 사용됩니다.
    - **`float: left`**는 왼쪽부터, **`float: right`**는 오른쪽부터 요소를 배치하게 합니다.
- **클리어 (Clear):**
    - 플로트가 끝난 후, 다음 요소가 플로팅된 요소의 영향을 받지 않고 원래 자리부터 시작하도록 **경계를 명확하게 구분**해주는 속성이 필요합니다.
- **위치 지정 (Positioning):**
    - 요소의 위치를 정하는 중요한 속성입니다. 기본값은 `static`입니다.
    - **`static`:** 별다른 위치 지정 속성을 주지 않았을 때의 기본 상태입니다.
    - **`relative`:** **자신을 기준으로** 위치를 이동시키면서, 동시에 다른 요소와 흐름을 유지하는 방식으로 배치됩니다.
    - **`absolute`:** 다른 요소들과의 연관성 없이, **부모 요소의 기준점**을 바탕으로 절대적인 위치에 배치됩니다.
    - **`fixed`:** 스크롤을 하더라도 화면에 **항상 고정되어** 움직이지 않는 위치에 배치됩니다 (예: 웹사이트의 고정 배너, 광고).

- 웹 페이지의 배치를 자유자재로 제어하는 것이 CSS의 핵심입니다. `padding`, `margin`, `float`, `position` 등 다양한 개념들이 서로 연관되어 있으므로, 개념을 이해하는 것이 가장 중요합니다.
- **가장 중요한 속성은 선택자(Selector)입니다.** 어떤 요소에 어떤 스타일을 적용할지 정확히 지정하는 능력이 가장 중요합니다.

## 1. CSS 선택자의 필요성 및 기본 개념
- CSS는 HTML 요소가 존재해야만 아무런 의미를 가질 수 있습니다. CSS의 목적은 곧 **HTML 요소를 꾸미거나 조작하는 것**입니다.
- **선택자(Selector)**는 CSS가 특정 HTML 요소를 찾아가게 하는 핵심 도구입니다.
- 선택자가 없다면 복잡하고 비효율적인 방법으로 요소를 찾아야 하지만, ID나 Class 같은 이름표를 붙여주면 원하는 요소를 매우 쉽고 효율적으로 접근할 수 있습니다.

## 2. 핵심 선택자 활용 (범용, ID, 클래스)
- **별표 선택자 (`*`):** 모든 HTML 요소를 선택합니다.
- **ID 선택자 (`#`):** 특정 요소에 부여된 ID를 이용하여 요소를 선택합니다.
    - `#` 기호를 사용하며, 페이지 전체에서 **유일해야 하는(Unique)** 목적의 식별자입니다.
    - 자바스크립트 등에서 요소를 하나만 찾을 때 이 특성이 중요합니다.
- **클래스 선택자 (`.`):** 특정 요소에 부여된 클래스명을 이용하여 요소를 선택합니다.
    - `.` 기호를 사용하며, 페이지 내 여러 요소에 동일하게 적용할 수 있어 **반복 사용**에 적합합니다.
- **ID와 클래스의 목적 차이:**
    - 두 선택자 모두 **'요소 찾기'**가 목적입니다.
    - **ID:** 해당 페이지에서 유일해야 하며, 단일 요소에 대한 강력한 식별자로 사용합니다.
    - **Class:** 스타일이나 속성을 여러 요소에 반복적으로 적용하고 싶을 때 사용합니다. 하나의 요소가 여러 클래스를 가질 수 있습니다.

## 3. 선택자 간의 우선순위 및 계층 구조
- **선택자의 우선순위 (Specificity):** 어떤 CSS 속성이 실제로 적용될지 결정하는 '가중치' 시스템입니다.
    - 일반 태그 선택자 (예: `p`, `div`)가 가장 낮은 가중치를 가집니다.
    - 클래스 선택자 (`.class`)는 일반 태그보다 높은 가중치를 가집니다.
    - ID 선택자 (`#id`)는 가장 높은 가중치(가장 강력함)를 가지므로, 같은 속성이 충돌할 경우 ID가 우선 적용됩니다.
- **속성 상속 (Inheritance):**
    - 부모 요소가 특정 속성(예: 배경색, 폰트)을 가지고 있을 경우, 자식 요소가 별도로 속성을 지정하지 않으면 부모의 속성이 상속되어 사용됩니다.
    - 자식 요소가 명시적으로 속성을 지정하면, 부모의 속성보다 우선하여 적용됩니다 (이 과정을 **오버라이딩**이라고 합니다).

## 4. CSS 조합 선택자 및 속성 선택자
- **조합 선택자 (Combinators):** 요소를 검색하는 구조적 관계를 나타냅니다.
    - **공백 (` `) - 자손(Descendant):** 먼저 지정된 요소 안에 있는 **모든** 후손 요소를 선택합니다 (가장 포괄적).
    - **`>` - 자식(Child):** 바로 하위 레벨에 있는 요소만을 선택합니다.
    - **`+` - 인접 형제(Adjacent Sibling):** 바로 앞선 형제 요소 **바로 다음에** 오는 요소만을 선택합니다.
    - **`~` - 일반 형제(General Sibling):** 같은 부모를 가지는 요소 중 특정 요소 **뒤에** 오는 모든 형제 요소들을 선택합니다.
- **속성 선택자 (`[속성명]`):** HTML 요소 자체의 속성값에 기반하여 요소를 선택합니다.
    - **속성명 지정:** 특정 속성을 가진 모든 요소를 선택할 수 있습니다 (예: `[title]`).
    - **속성값으로 필터링:** 속성의 값이 특정 조건을 만족하는 요소를 선택할 수 있습니다.
        - `[속성*="값"]`: 속성값에 '값'이 포함된 모든 요소를 선택합니다.
        - `[속성^="값"]`: 속성값이 '값'으로 시작하는 요소를 선택합니다.
        - `[속성$=...]`: 속성값이 '값'으로 끝나는 요소를 선택합니다.

## 5. 특수 선택자 (가상 클래스 및 조합자)
- **가상 클래스 (Pseudo-class):** 요소가 가지는 '상태'를 기준으로 선택하는 선택자입니다.
    - `a:link`: 아직 방문하지 않은 일반 링크
    - `a:visited`: 이미 방문했던 링크
    - `a:hover`: 마우스 커서가 올라온 상태
    - `a:active`: 현재 클릭하고 있는 상태
    - 이 외에도 `:nth-child()`, `:first-child`, `:last-child` 등 다양한 상태를 선택할 수 있습니다.
- **전체 요약:** CSS와 자바스크립트가 작동하는 근본적인 목적은 **HTML 요소를 효율적으로 찾아내는 것**이며, ID, Class, 조합 선택자, 가상 클래스, 속성 선택자 등은 모두 이 '찾아가는 과정'을 편리하게 도와주는 장치입니다.
