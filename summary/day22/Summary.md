## 1. 오리엔테이션 및 휴가 이야기
- **강의 시작:** 출석 확인 및 참여 독려로 강의를 시작했습니다.
- **휴가 후 근황 공유:** 수강생들과 휴가 경험(여행, 휴식)을 공유하며 가볍게 소통했습니다.
- **학습 태도 점검:** 지나간 것에 대한 후회는 접어두고, 오늘부터 해야 할 학습 계획을 차곡차곡 진행하는 것이 중요함을 강조했습니다.

## 2. 웹 개발의 기본 구조와 자바의 위치
- **학습의 연속성:** 자바(Java)는 디자인 패턴을 비롯하여 전반적인 심화 학습이 필요하며, 이는 꾸준히 이어갈 내용임을 강조했습니다.
- **Java의 중요성:** 우리 과정은 전반적으로 자바를 중심으로 진행되며, 관련된 개념들을 놓치지 않고 학습해야 합니다.
- **클라이언트-서버 구조 이해 (프론트/백엔드):**
    - **백엔드 (Java):** 데이터 가공 및 처리를 담당하는 강력한 언어입니다. (Java가 애초에 애플릿 기술로 시작되었으나, 느리다는 단점 때문에 백엔드 영역에서 빠르게 발전함.)
    - **프론트엔드 (HTML/CSS/JS):** 사용자에게 데이터를 보여주고 인터페이스를 구성하는 역할을 합니다.
    - **상호작용:** 웹 애플리케이션은 백엔드와 프론트엔드가 데이터를 주고받으며 협력해야 완성됩니다. 백엔드 개발자 역시 프론트엔드가 어떻게 보여질지 이해하는 것이 중요합니다.

## 3. HTML의 역사와 웹 표준의 발전
- **HTML의 본질:** HTML은 **하이퍼텍스트 마크업 랭귀지(Hypertext Markup Language)**의 약자입니다.
- **마크업 언어와 하이퍼텍스트:**
    - **마크업 언어:** XML처럼 태그를 사용하여 콘텐츠의 구조와 약속을 정하는 언어입니다.
    - **하이퍼텍스트:** 페이지 간의 링크(Link)를 통해 명령 없이 문서를 연결하여 이동하는 개념입니다. HTML의 도입으로 현재의 인터넷 환경이 만들어졌습니다.
- **웹 브라우저의 변화 (브라우저 전쟁):**
    - **초기:** 네스케이프의 모자이크(Mosaic)가 최초의 브라우저로 등장했습니다.
    - **문제점:** 초기 브라우저들(Internet Explorer, Chrome, Safari 등)이 문서 해석 방식이 달라 개발자들이 매우 어려움을 겪었습니다.
    - **결과:** 표준화 요구가 높아지면서 경쟁과 진화가 이루어졌으며, 이는 웹 개발의 기준을 정립하는 계기가 되었습니다.

## 4. HTML5의 핵심 구조와 동작 원리
- **HTML5의 등장:** 2016년경 HTML5가 등장하며 웹 개발에 큰 변화를 가져왔습니다.
- **DOCTYPE의 역할:** HTML 문서의 가장 첫 줄에 나오는 `<!DOCTYPE html>`은 이 문서가 어떤 표준(예: HTML5)을 기준으로 해석되어야 하는지 브라우저에게 알려주는 역할을 합니다.
- **HTML 구조의 이해 (XML과의 비교):**
    - HTML은 XML처럼 태그로 구조를 정의합니다.
    - **DOM (Document Object Model):** HTML 태그들이 트리(Tree) 형태로 그려질 수 있도록 구조화된 모델을 의미합니다.
    - **HTML Root Element 구조:** 모든 HTML 문서의 최상위 요소는 `<html`입니다. 이 요소는 자손으로 `<head>`와 `<body>`라는 두 개의 엘리먼트만을 가집니다.
        - **`<head>`:** 웹페이지 전체에 대한 메타 정보(제목, 작성자, 스타일 시트 링크 등)를 담는 영역으로, 사용자에게 직접 보이는 내용은 없습니다.
        - **`<body>`:** 사용자의 눈에 실제로 보이는 모든 콘텐츠가 담기는 영역입니다.
- **HTML5의 혁신:** HTML5는 단순한 마크업(구조)을 넘어서 `DOM API`와 같은 새로운 API를 포함하여 프로그래밍적인 기능까지 할 수 있게 되었습니다.

## 5. JavaScript 실행 환경 이해 (Node.js)
- **JavaScript의 실행 환경의 한계:**
    - **기존 (브라우저 환경):** 자바스크립트는 원래 웹 브라우저가 제공하는 개발자 도구(Console) 내에서 실행되었습니다. 브라우저가 자바스크립트를 해석하고 실행하는 도구(Engine)를 갖고 있기 때문입니다.
    - **한계:** 이 때문에 자바스크립트는 기본적으로 브라우저 내에서만 동작하는 한계가 있었습니다.
- **Node.js의 등장 (브라우저 밖):**
    - **필요성:** 자바스크립트를 브라우저 밖(서버 환경)에서도 실행할 수 있도록 도와주는 도구가 필요했습니다.
    - **Node.js의 역할:** Node.js는 이 외부 실행 환경을 제공하는 도구(런타임)입니다. (JDK가 자바를, Python이 파이썬을 실행하는 것처럼, Node.js는 자바스크립트를 독립적으로 실행하게 합니다.)
- **Node.js 버전 관리의 중요성:**
    - 여러 프로젝트는 다른 버전의 Node.js를 요구할 수 있습니다.
    - **NVM (Node Version Manager):** NVM은 Node.js 버전 관리자 역할을 합니다. 이를 통해 여러 버전의 Node.js를 쉽게 설치하고 필요에 따라 버전을 전환하며 사용할 수 있게 해줍니다.

## 1. Node.js 환경 설정 및 버전 관리 (NVM)
- **Node.js 필요성:** 웹 페이지를 브라우저 외 환경(서버 등)에서 자바스크립트로 동작시키기 위해 Node.js가 필수적임.
- **NVM의 역할:** Node Version Manager (NVM)를 이용하면 다양한 버전의 Node.js를 손쉽게 설치하고 관리할 수 있어, 특정 버전을 테스트하거나 사용할 때 매우 편리함.
    - **주요 명령어:**
        - `nvm list`: 설치된 Node.js 버전 리스트 확인.
        - `nvm install <버전> `: 원하는 버전의 Node.js를 다운로드 및 설치.
        - `nvm use <버전>`: 현재 사용하려는 Node.js 버전 지정.
        - `nvm uninstall <버전>`: 특정 버전 삭제.
- **프로젝트 초기화:** Node 프로젝트를 시작할 때 `npm init` 명령어를 사용하여 프로젝트를 초기화하며, 이 과정에서 `package.json` 파일이 생성됨.
    - `package.json` 파일: 해당 Node 프로젝트가 필요한 기본 정보(의존성 등)를 기술하는 파일.

## 2. 웹 서버의 개념 이해 및 구축
- **서버-클라이언트 모델 이해:** 웹 애플리케이션은 클라이언트(브라우저)가 요청을 보내면, 서버가 해당 요청을 받아 적절한 정보(HTML 파일 등)를 응답 형태로 되돌려주는 방식으로 동작함.
- **웹 서버의 역할 (Light Server 예시):**
    - 웹 서버는 단순하게 요청(Request)을 받아서 요청된 파일(Resource)을 찾고, 그 파일을 그대로 응답(Response)으로 돌려주는 역할만을 함.
    - **포트 번호(Port):** 서버가 동작하는 통로 번호. 일반적으로 `localhost:3000`과 같이 사용되며, 서버가 동작하는 방(Port)의 개념으로 이해함.
    - **요청 처리 방식:** 특정 파일을 요청할 때, 서버는 기본적으로 `index.html` 파일을 찾거나, 지정된 경로(`hello.html`, `dir/file.html`)의 파일을 찾아 응답함.
- **HTTP 상태 코드 이해:** 웹 통신에서 서버와 클라이언트 간에 주고받는 신호입니다.
    - **200 (성공):** 요청이 성공적으로 처리됨.
    - **400번대 (클라이언트 오류):** 클라이언트 측에서 요청 자체가 잘못되었음을 의미함.
    - **404 (찾을 수 없음):** 요청된 리소스(페이지)가 서버에 존재하지 않음을 의미함.
    - **304 (Not Modified):** 이미 받은 리소스에 변경 사항이 없을 때, 데이터 전송 없이 "이전 파일을 그대로 사용해도 좋다"고 알려주는 효율적인 응답 코드.
- **서버 동작의 중요성:** 웹 페이지를 파일 탐색기에서 여는 것이 아니라 서버를 동작시키고 서버에게 요청을 보내야만 올바르게 페이지가 로드되고 내용이 변하는 등의 동작을 할 수 있음.

## 3. HTML의 구조와 의미론적 태그 활용
- **HTML의 기본 구조:** HTML은 마크업 언어(Markup Language)로, 태그(Tag)라는 약속을 기반으로 문서를 구성함.
- **문서의 필수 요소:**
    - **`<!DOCTYPE html>`:** 이 문서가 어떤 버전의 HTML(현재는 HTML5)을 따르는지 브라우저에게 알려주는 선언문.
    - **`<html>`:** 모든 HTML 요소의 최상위 루트(Root) 요소. 모든 내용이 이 태그 안에 포함되어야 함.
    - **`<head>`:** 페이지 자체에 대한 메타데이터(Meta Data)를 담는 공간. 사용자에게 직접 보이지는 않으나, 브라우저나 검색 엔진에게 정보를 제공함.
        - **`title`:** 브라우저 탭에 표시되는 문서의 제목.
    - **`<body>`:** 실제 웹 페이지에 사용자에게 보여지는 모든 내용이 들어가는 본문 영역.
- **태그의 구성 요소:**
    - **시작 태그 (`<tag>`) - 종료 태그 (`</tag>`) - 콘텐츠:** 대부분의 HTML 요소는 시작 태그와 종료 태그 사이에 콘텐츠가 포함되는 구조(엘리먼트)를 가짐.
    - **속성 (Attribute):** 태그에 추가적인 정보를 제공하는 키-값 쌍 (예: `<img src="경로" alt="설명">`에서 `src`, `alt`가 속성).
    - **자가 종료 태그 (Self-closing Tag):** 시작과 종료 태그가 없이 자기 스스로를 닫는 태그 (예: `<br>`, `<hr>`).
- **의미론적 태그와 SEO (Search Engine Optimization):**
    - **헤딩 태그 (`<h1>` ~ `<h6>`):** 글씨 크기가 크다고 중요도가 높은 것이 아니라, 검색 엔진이 문서의 중요도를 판단하는 핵심 키워드로 인식하는 "의미"를 부여함.
    - **중요성:** `<h1>` 태그를 사용할 때는 단순히 크게 보여주기 위해서가 아니라, 해당 내용이 문서의 가장 중요한 제목임을 검색 엔진에 알려주기 위함. (이는 검색 엔진 최적화(SEO)에 매우 중요함.)
    - **적용 원칙:** 본문 내용을 구성할 때, 태그를 선택할 때 내용의 의미에 맞는 태그를 사용해야 함.
- **주석 (Comment):** `<!-- 내용 -->` 형태로 작성하며, 이는 브라우저나 서버가 해석하지 않고 무시하는 주석 정보임.

## 1. HTML의 기본 구조 이해 (요소, 콘텐츠, 속성)
- **요소(Element)의 구조:** 모든 HTML 태그는 시작 태그와 종료 태그로 구성되는 '엘리먼트'로 이루어집니다.
- **태그 구성:** `<태그명> 콘텐츠 </태그명>`의 구조를 가집니다.
- **콘텐츠:** 태그 안에 들어가는 내용만을 의미하는 것이 아니라, 웹 페이지의 모든 내용(Body 전체)을 포괄적으로 지칭합니다.
- **셀프 클로징 태그:** 내용을 갖지 않고 기능을 수행하는 태그 (예: `<hr>`, `<br>`)는 시작 태그와 종료 태그가 없는 '셀프 클로징 태그'로 표현됩니다.
- **속성(Attribute):** 태그가 가질 수 있는 추가적인 정보를 정의하며, 일반적으로 `속성 이름="속성 값"` 형태로 사용됩니다. (예: `<img src="경로" alt="설명">`)
- **주석(Comment):** `<!-- 주석 내용 -->` 형태로 작성하며, 브라우저가 해석하지 않으므로 화면에 출력되지 않아 코드 설명을 위해 사용합니다.

## 2. HTML의 핵심 구성 요소와 구조화
- **기본 구조:** 표준 HTML 문서는 `<html lang="언어코드">` 태그 안에 `<head>`와 `<body>`로 크게 나뉩니다.
- **`<head>` 태그 (메타 정보):**
    - 브라우저가 문서를 해석할 때 필요한 정보(메타 정보)를 담습니다.
    - 필수 메타 태그로는 문자 인코딩 설정, 뷰포트(반응형 웹 관련) 등이 있습니다.
    - **SEO 메타 태그:** 검색 엔진 최적화(SEO)를 위해 페이지에 대한 설명, 키워드 등의 정보를 제공합니다. 적절한 메타 태그를 사용하면 검색 노출 시 가중치를 높이는 데 도움이 됩니다.
- **`<h1>`와 `<p>` 태그의 중요성:**
    - `<h1>` 등의 제목 태그는 단순히 글씨를 크게 보여주는 용도 외에, 검색 엔진이 콘텐츠의 **제목의 중요도(가중치)**를 파악하는 중요한 요소입니다.
    - `<p>` 태그는 본문의 실제 내용을 담을 때 사용합니다.
- **스타일 분리 원칙:** 굵게, 기울임꼴, 크기 조절 등 **시각적인 스타일(Style)**은 태그 자체보다는 CSS를 사용하여 분리하여 적용하는 것이 좋습니다.

## 3. 웹 접근성을 고려한 요소 사용 (이미지, 링크)
- **하이퍼링크 (`<a>` 태그):**
    - 하이퍼텍스트의 핵심 요소로, `<a>` 태그와 `href` 속성을 사용하여 문서를 다른 문서로 이동시킬 수 있습니다.
- **이미지 (`<img>` 태그):**
    - 이미지를 삽입할 때 `src` 속성으로 파일 경로를 지정합니다.
    - **Alt 속성의 중요성 (웹 접근성):** `alt` 속성은 이미지의 대체 텍스트를 제공합니다. 이 속성은 시각 장애인이 사용하는 스크린 리더(Screen Reader)가 이미지를 설명해주기 때문에, 콘텐츠의 의미를 전달하는 데 매우 중요합니다.
    - 이미지 로딩 실패 시에도 대체 텍스트로 보여지거나, 검색 엔진이 이미지를 이해하는 데 도움을 줍니다.
- **목록 및 정의 목록:**
    - 순서가 있는 목록은 `<ol>`, 순서가 없는 목록은 `<ul>`을 사용하며, 정의 목록은 `<dl>` 태그를 이용합니다.
    - **참고:** 목록이나 테이블의 디자인(순서, 모양 등)은 CSS로 얼마든지 수정할 수 있습니다.

## 4. 콘텐츠 구조화 및 의미 부여 (리스트, 테이블, 시맨틱 태그)
- **테이블 (`<table>`):**
    - 테이블은 단순히 데이터를 보여주는 영역을 넘어, **데이터 형태**의 정보를 표시하는 데 사용합니다.
    - **Semantic (의미론적) 태그의 이해:** `<thead>` (테이블 헤더), `<tbody>` (테이블 본문) 등의 태그는 시각적으로는 큰 차이가 없을 수 있지만, 해당 영역이 *'무엇의 의미를 담고 있는지'*를 구조적으로 정의하여 기계(프로그램, 검색 엔진)가 이해하도록 돕습니다.
    - 이는 데이터를 프로그램적으로 처리하거나 동적으로 만들어낼 때 필수적인 역할을 합니다.
- **HTML5의 발전:**
    - HTML5에서는 웹페이지의 의미를 더욱 명확히 전달할 수 있는 다양한 시맨틱 태그들이 도입되었습니다.

## 5. 웹 통신과 데이터 전송 이해 (Form, HTTP)
- **폼 태그 (`<form>`):**
    - 폼은 사용자의 데이터를 입력받아 서버와 통신할 수 있게 하는 '통로' 역할을 합니다.
    - **`action` 속성:** 폼 제출(Submit) 버튼을 눌렀을 때 데이터가 전송될 목적지(URL)를 지정합니다.
    - **`type="submit"`:** 버튼의 역할을 정의하며, 이 버튼이 클릭되면 폼 전송 이벤트를 발생시킵니다.
    - **`name` 속성:** 폼 입력 필드에 사용되는 필수 속성입니다. 서버로 데이터가 전송될 때, 이 `name` 속성이 **'키(Key)'**가 되어 서버가 해당 **'값(Value)'**을 식별하고 처리할 수 있게 합니다.
- **클라이언트-서버 통신 원리:**
    - 웹 서비스는 클라이언트(사용자의 브라우저)와 서버(정보를 가지고 있는 컴퓨터 시스템)가 네트워크를 통해 상호 작용합니다.
    - **HTTP 통신 규약:** 클라이언트와 서버가 데이터를 주고받기 위해 사용하는 정해진 통신 규약입니다.
    - **통신 과정:**
        1. 클라이언트가 주소와 포트번호를 이용해 서버에 접속을 요청합니다.
        2. 클라이언트가 원하는 리소스(예: `index.html`)를 요청합니다.
        3. 서버는 요청을 받아 해당 내용을 응답(Response) 형태로 돌려줍니다.
    - **Query String (쿼리 문자열):**
        - URL에 붙는 `?키=값&키2=값2` 형태의 문자열입니다.
        - 클라이언트가 서버에 데이터를 요청하며 함께 전달하는 값이 들어가는 부분입니다. 서버는 이 정보를 읽어 페이지를 동적으로 구성하고 동작에 활용합니다.

## 6. 학습 참고 자료 안내

## 1. 웹 페이지의 기본 구조 이해 (Client-Server 및 HTML)
- **웹 페이지의 구성 요소:**
    - 웹 페이지는 결국 HTML이라는 언어로 작성된 문서 형태로 시작됩니다.
    - **백엔드(Backend):** 데이터 가공 및 데이터를 전송하는 역할을 담당합니다.
    - **프론트엔드(Frontend):** 서버로부터 받은 데이터를 화면에 보기 좋게 보여주는 역할을 담당합니다.
    - **작동 원리:** 서버가 HTML 문서(텍스트)를 클라이언트(브라우저)로 전달하면, 브라우저는 이 문서에 포함된 태그(Tag)와 CSS 등의 규칙에 따라 실제 화면을 해석하여 보여줍니다.
- **웹 문서의 특징:**
    - 모든 웹페이지는 HTML 문서이며, 이 문서는 마치 워드 프로세스로 문서를 만드는 것과 유사하게 구조화됩니다.
    - **링크(Hyperlink):** `<a>` 태그와 `href` 속성을 이용하여 다른 문서와 연결하는 역할을 합니다.
    - **URL 구조의 이해:** URL을 관찰하며 누가, 어떤 경로에 있는 페이지를 요청하는지, 그리고 매개변수(Parameter)를 통해 값을 전달하는 방식을 이해해야 합니다.
    - **동적 콘텐츠:** 실제 뉴스 기사나 웹 콘텐츠처럼 내용은 같더라도 URL에 전달된 값(파라미터)에 따라 서버가 데이터베이스(DB)에서 내용을 읽어와 페이지를 *동적으로* 완성하여 보여줄 수 있습니다.

## 2. 클라이언트와 서버의 통신 방식 (HTTP 프로토콜 및 데이터 요청)
- **클라이언트-서버 구조:**
    - **클라이언트(Client):** 정보를 요청하는 주체 (사용자, 브라우저 등).
    - **서버(Server):** 정보를 제공하는 주체.
    - **작동 흐름:** 클라이언트가 서버에 접속을 시도하고(요청), 서버가 그 요청을 받아 응답(Response)을 하는 방식으로 작동합니다.
- **HTTP 프로토콜의 특징:**
    - 웹은 HTTP(Hypertext Transfer Protocol)라는 프로토콜 위에서 동작합니다.
    - **Stateless(무상태성):** HTTP는 기본적으로 **접속이 요청-응답이 끝나면 종료**됩니다. 즉, 서버는 이전의 접속 상태를 기억하지 않습니다.
    - **요청-응답 반복:** 사용자가 링크를 누를 때마다, 접속하고 요청하고 응답받은 후 다시 접속이 종료되는 과정이 반복됩니다.
- **요청 방식 (GET vs. POST):**
    - **GET (기본값):** 요청 데이터를 URL의 쿼리 스트링(Query String, 물음표 이후의 값) 형태로 전송합니다. (ex: `...?key=value`)
    - **POST:** 데이터를 URL에 노출시키지 않고, 몸체(Body) 안에 담아서 서버로 전송합니다.
    - **활용 판단:** 전송하려는 데이터가 노출되어도 안전한지, 데이터의 길이가 긴지 등을 고려하여 GET과 POST를 선택해야 합니다.

## 3. 사용자 입력의 핵심, 폼(Form) 태그 활용
- **폼 태그의 역할:** 웹에서 사용자가 정보를 입력하고 제출(Submit)할 때 사용되는 핵심 태그입니다.
- **작동 메커니즘:**
    - 폼을 제출하면 설정된 `action` 경로로 이동하며, 폼 안에 있는 각 입력 필드(`input`)의 값이 `name` 속성을 통해 서버로 전송됩니다.
    - **`<label>` 태그:** 입력 필드와 설명을 연결하여, 접근성 및 사용자 편의성을 높이는 데 사용됩니다.
- **폼의 종류 및 속성:**
    - **`method` 속성:** 데이터를 GET 또는 POST 방식으로 보낼지 결정합니다. (일반적으로 기본값은 GET입니다.)
    - **HTML5 Input Types:** 단순한 텍스트 박스 외에도 데이터의 형태에 맞게 전문화된 `type` 속성을 사용할 수 있습니다.
        - `text`: 일반 텍스트 (가장 많이 사용됨)
        - `password`: 입력 값이 별표(*)로 가려져서 표시됨.
        - `email`, `url`: 해당 형식의 유효성 검사가 기본적으로 수행됨.
        - `number`: 숫자 입력만 가능하며 범위 제한 설정 가능.
        - `date`: 달력(Date Picker)이 자동 생성되어 날짜 선택이 편리함.
        - **`required`:** 해당 필드는 반드시 값을 입력해야만 제출 버튼이 활성화되도록 강제합니다.
- **추가 요소:**
    - **`radio` 버튼:** 여러 선택지 중 오직 하나만 선택 가능하도록 묶여야 합니다.
    - **`checkbox`:** 여러 개를 중복해서 선택할 수 있습니다.
    - **`textarea`:** 여러 줄에 걸쳐 긴 텍스트를 입력받을 때 사용합니다.
    - **`select`:** 드롭다운 메뉴를 만들어 사용자에게 옵션을 제공할 때 사용합니다.

## 4. 웹 구조의 의미 부여: 시맨틱 태그(Semantic Tags)
- **시맨틱 태그란?**
    - 특별한 기능(Functionality)을 수행하는 것이 아니라, **'의미(Meaning)'**를 부여하기 위해 존재하는 태그들입니다.
    - (예: `<div>` 태그는 의미가 없어 단순히 묶기만 하지만, `<header>` 태그는 '이 부분이 페이지의 머리글 영역이다'라는 의미를 가집니다.)
- **주요 시맨틱 태그:**
    - `<header>`: 웹 페이지의 머리글 영역.
    - `<nav>`: 웹사이트의 주 메뉴/탐색(Navigation) 영역.
    - `<main>`: 페이지의 핵심 콘텐츠 영역.
    - `<section>`: 주제별로 내용을 구분하는 영역.
    - `<article>`: 독립적인 기사나 콘텐츠 단위.
    - `<footer>`: 웹 페이지의 바닥글 영역.
- **사용의 중요성:**
    - **관리의 용이성:** 구조적으로 묶어 관리하기가 훨씬 편리합니다. (예: 헤더 영역 전체를 통째로 옮기기 용이).
    - **검색 엔진 최적화 (SEO):** 검색 엔진이 웹페이지의 내용을 단순히 텍스트로 읽는 것이 아니라, 구조와 의미를 파악하여 페이지의 중요도를 판단하는 데 매우 유리합니다.
    - **접근성:** 시각 장애인이나 스크린 리더와 같은 보조 장치를 사용하는 사람들에게 웹페이지의 구조를 정확하게 전달해주는 중요한 역할을 합니다.

## 1. HTML의 역할: 웹페이지의 뼈대와 구조
HTML(HyperText Markup Language)은 웹페이지가 보여줄 내용(텍스트, 이미지 등)의 **뼈대(구조)**를 만드는 역할을 합니다.

*   **구조적 역할:** 정보를 논리적으로 담는 그릇을 제공하며, 시멘틱 태그(Semantic Tag)를 활용하여 문서의 의미를 명확하게 분리하는 것이 중요합니다.
    *   예: `<header>`, `<nav>`, `<main>`, `<footer>` 등의 태그를 사용하면 구조를 명확히 할 수 있습니다.
*   **구조화의 중요성:** 웹 접근성(Accessibility)과 검색 엔진 최적화(SEO) 측면에서 내용의 의미를 명확하게 구분하는 것이 매우 중요합니다.
*   **미디어 지원:** HTML 태그는 텍스트뿐만 아니라 비디오(Video), 오디오(Audio) 등 다양한 멀티미디어 요소를 포함할 수 있게 합니다.

## 2. CSS의 역할: 디자인과 시각적 스타일링
CSS(Cascading Style Sheet)는 HTML로 만들어진 **뼈대**를 실제로 어떻게 보이게 할지, 즉 **스타일(Style)**을 정의하는 역할을 합니다.

*   **역할:** HTML은 내용 그 자체에 집중하고, CSS는 색상, 글꼴 크기, 배치, 레이아웃 등 시각적인 표현을 담당합니다.
*   **CSS의 시각적 힘:** 내용(HTML)이 동일하더라도, CSS를 어떻게 적용하느냐에 따라 페이지의 모습과 사용자 경험이 완전히 달라질 수 있습니다.
*   **웹 구조의 목표:** HTML은 내용(Content)만을 담고, CSS는 디자인(Presentation)을 전담하도록 역할을 분리하는 것이 가장 바람직합니다.

## 3. HTML과 CSS의 분리 원칙과 CSS 적용 방법
CSS를 효과적으로 사용하기 위해서는 HTML과 CSS를 분리하여 관리하는 것이 원칙이며, 이는 유지보수와 일관된 디자인 적용에 용이합니다.

*   **권장 방식 (외부 스타일시트):** 별도의 `.css` 파일을 만들어 HTML 문서와 연결하는 방식이 가장 이상적입니다. (가장 많이 사용됨)
*   **다른 적용 방식:**
    *   **인라인 스타일 (Inline Style):** HTML 태그의 속성(`style=""`)에 직접 스타일을 적용하는 방법입니다. (권장하지 않음)
    *   **내부 스타일시트 (Internal Style):** HTML 문서의 `<head>` 태그 안에 `<style>` 태그를 사용하여 작성하는 방법입니다.
*   **CSS를 적용하는 원리:** CSS가 어떤 요소를 꾸미려면, 해당 요소가 무엇인지 **명확히 선택**해 주어야 합니다.

## 4. CSS의 핵심 요소: 선택자(Selector)와 박스 모델(Box Model)
CSS를 사용하여 원하는 스타일을 적용하기 위해 반드시 이해해야 하는 핵심 개념들입니다.

### 📄 선택자 (Selector)의 중요성
선택자는 CSS가 스타일을 적용할 대상을 지정하는 과정입니다.

*   **선택자 정의:** 특정 HTML 태그(예: `p`, `h1`), 특정 클래스, 특정 ID를 '찾아내는 것'이 선택자입니다.
*   **핵심:** CSS, JavaScript 등 모든 웹 기술의 존재 목적은 결국 HTML을 찾아(선택자) 그 내용을 다루거나(JavaScript) 꾸미기(CSS) 위함이므로, **선택자 사용법을 잘 익히는 것이 가장 중요**합니다.
*   **타입 선택자:** 가장 기본적인 형태로, 태그 이름 자체를 기준으로 스타일을 적용합니다 (예: `p` 태그의 모든 요소).

### 📦 박스 모델 (Box Model) 이해하기
HTML의 모든 태그는 실제로 상자(Box) 형태로 존재하며, 이 상자 구조를 이해하는 것이 CSS의 기본입니다.

1.  **박스(Box):** 모든 HTML 요소는 테두리를 가진 상자 형태입니다.
2.  **콘텐츠 (Content):** 사용자가 실제로 보는 내용물 영역입니다.
3.  **패딩 (Padding):** 콘텐츠와 테두리(Border) 사이의 **내부 여백**입니다.
4.  **테두리 (Border):** 박스의 외곽선을 정의하는 선의 굵기입니다.
5.  **마진 (Margin):** 박스(요소)와 다른 박스 사이의 **외부 여백**입니다.

> **💡 기억할 점:** CSS를 사용할 때 발생하는 레이아웃 문제(예: 간격이 이상한 경우)는 요소 자체에 기본으로 설정된 마진이나 패딩 값 때문일 수 있으므로, 필요에 따라 초기화(Reset)해 주는 것이 좋습니다.

## 1. CSS 및 HTML 기본 구조 이해 (박스 모델과 초기 설정)
- **모든 HTML 요소는 박스(Box) 형태:** 모든 HTML 태그는 기본적으로 네모 박스로 구성되어 있으며, 이 박스들이 중첩되어 구조를 이룹니다.
- **Box Model의 이해:**
    - **테두리(Border):** 박스의 경계선입니다.
    - **패딩(Padding):** 테두리 안쪽, 실제 콘텐츠와 테두리 사이의 간격입니다.
    - **마진(Margin):** 박스와 다른 박스 사이의 간격입니다.
- **초기화(Reset)의 중요성:** 브라우저마다 기본 설정값(패딩, 마진 등)이 다르기 때문에, 개발 시에는 CSS를 사용하여 전역적으로 마진과 패딩을 초기화(Reset)하여 일관된 환경을 만드는 것이 중요합니다.

## 2. CSS 선택자 활용 방법 (Selector: Type, Class, ID)
CSS 선택자는 어떤 요소를 스타일링할지 결정하는 방법입니다. 선택자에는 각각 역할과 우선순위(Specificity)가 있습니다.

- **① 타입 선택자 (Type Selector):**
    - **사용법:** 태그 이름 자체를 사용합니다 (예: `p`, `div`).
    - **특징:** 해당 태그가 페이지 전체에 적용됩니다. (예: 페이지의 모든 `<p>` 태그에 스타일 적용).
    - **우선순위:** 가장 낮은 가중치를 가집니다.
- **② 클래스 선택자 (Class Selector):**
    - **사용법:** 앞에 마침표(`.`)를 붙이고 클래스 이름을 사용합니다 (예: `.highlight`).
    - **특징:** 특정 속성(Attribute)으로 부여한 이름으로 요소를 지정합니다. 여러 요소가 같은 클래스를 공유할 수 있어, 웹 전체의 **톤앤매너(Theme)**를 일관되게 유지할 때 매우 유용합니다.
    - **사용성:** 하나의 HTML 태그에 여러 클래스 속성을 동시에 부여할 수 있습니다.
- **③ ID 선택자 (ID Selector):**
    - **사용법:** 앞에 샵 기호(`#`)를 붙이고 ID 이름을 사용합니다 (예: `#header`).
    - **특징:** 페이지 내에서 **반드시 한 번만** 사용해야 하는 고유한 요소에 사용됩니다. (유일성 전제)
    - **용도:** 웹 페이지의 특정 구조를 JavaScript로 제어할 때 주로 사용됩니다.
- **선택자 우선순위 (Specificity):**
    - 선택자는 가중치(Weight)를 가집니다. 가중치가 높은 선택자가 우선 적용됩니다.
    - **가중치 순서 (높은 순):** ID 선택자 > 클래스 선택자 > 타입 선택자.
    - **동일 가중치일 때:** 나중에 정의된 CSS 규칙이 우선 적용됩니다.

## 3. CSS 스타일링 속성 및 단위 (Color, Size, Text)
- **색상값 지정:**
    - CSS에서 색상은 16진수(Hex: `#RRGGBB`), RGB 값, 또는 색상 이름(예: `red`)으로 지정할 수 있습니다.
    - *실무 참고:* 구체적인 색상 코드는 디자이너로부터 받는 것이 일반적입니다.
- **단위(Unit)의 종류:**
    - **픽셀 (px):** 화면의 점(Pixel) 크기를 기준으로 하는 **고정 단위**입니다. 화면 크기에 관계없이 고정된 크기로 표시됩니다.
    - **상대 단위 (em, rem 등):** 브라우저의 기본 폰트 크기나 부모 요소의 크기를 기준으로 상대적인 크기를 지정합니다. **반응형 웹** 디자인 시, 화면 크기에 따라 유연하게 변하도록 설정할 때 매우 바람직합니다.
- **텍스트 관련 속성:**
    - `text-align`: 텍스트의 가로 정렬 (좌측, 중앙, 우측).
    - `letter-spacing`: 글자(문자)와 문자 사이의 간격 조절.
    - `word-spacing`: 단어와 단어 사이의 간격 조절.
    - `text-indentation`: 문단 첫 줄의 들여쓰기 설정.

## 4. 레이아웃을 위한 Display 속성 이해 (Block, Inline, Inline-Block)
모든 HTML 태그는 기본적으로 박스 형태로 존재하며, 이 박스가 어떤 레벨(Level)을 가지는지에 따라 레이아웃이 결정됩니다.

- **블록 레벨 (Block Level):**
    - **특징:** 콘텐츠의 크기와 상관없이 무조건 **줄 전체(100%)**를 차지합니다. (예: `<p>`, `<div>`, `<h1>`)
    - **결과:** 항상 새로운 줄에서 시작합니다.
- **인라인 레벨 (Inline Level):**
    - **특징:** 콘텐츠가 차지하는 **최소한의 공간만** 차지합니다. (예: `<span>`, `<a>`)
    - **결과:** 같은 줄에 나란히 배치됩니다.
- **인라인 블록 (Inline-Block):**
    - **특징:** 인라인 레벨의 배치 특성(같은 줄에 나란히 배치)을 가지면서도, **블록 레벨처럼 너비(width)와 높이(height)**를 지정할 수 있게 하는 하이브리드 속성입니다.
    - **활용:** 웹에서 레이아웃을 구성할 때, 요소를 옆으로 배치하면서 원하는 크기로 제어하고 싶을 때 가장 핵심적으로 사용됩니다.

## 5. 종합 정리 및 학습 마인드셋

- **HTML의 본질:** 모든 HTML은 결국 박스(Box)로 이루어져 있으며, 상위-하위 관계가 트리 구조(DOM Tree)를 형성합니다.
- **CSS의 역할:** CSS는 구조화된 HTML 박스들에 시각적인 스타일(색상, 간격, 배치 등)을 적용하는 역할을 합니다.
- **핵심 개념의 이해:** 웹 개발 시, 특정 구문(태그, 클래스, ID 등)을 외우는 것보다 **HTML이 어떻게 구성되고(구조), CSS가 그 구조에 어떻게 배치와 스타일을 입히는지(흐름)**에 대한 전반적인 원리 이해가 가장 중요합니다.
- **궁극적 목표:** HTML과 CSS가 어떻게 상호작용하여 웹 페이지를 시각화하고 배치하는지에 대한 큰 그림을 파악하는 것이 중요합니다.


1. 클라이언트-서버 구조

클라이언트는 요청하는 쪽이고 서버는 요청을 받아 응답하는 쪽이다.
웹에서는 보통 브라우저가 클라이언트이고 Spring Boot/Django 같은 백엔드 프로그램이 서버 역할을 한다.
클라이언트는 직접 데이터를 가지고 있다기보다 서버에게 필요한 데이터를 요청해서 받아온다. 예를 들어 사용자가 게시글 목록 페이지에 들어가면 브라우저가 서버에 게시글 목록을 요청하고 서버는 DB에서 게시글을 가져와 HTML이나 JSON 형태로 응답한다.

2. HTTP 요청/응답

HTTP는 클라이언트와 서버가 데이터를 주고받는 약속이다.
클라이언트가 이 페이지 주셈 하고 요청하면 서버가 HTML, JSON, 이미지 같은 데이터를 응답으로 돌려준다.
HTTP 요청에는 보통 요청 주소, 요청 방식, 추가 데이터가 들어간다. 서버의 응답에는 상태 코드와 데이터가 들어간다.

3. GET / POST

GET은 데이터를 URL에 붙여서 보내는 방식으로, 검색이나 조회에 많이 쓴다.
GET은 URL에 값이 보이기 때문에 검색 조건이나 페이지 번호처럼 공유해도 되는 값에 적합하다.
POST는 데이터를 요청 본문에 담아서 보내는 방식으로, 로그인, 회원가입, 글쓰기처럼 서버에 데이터를 저장하거나 변경할 때 많이 쓴다.
POST는 URL에 값이 직접 보이지 않고 요청 body에 담겨 전송된다. 그래서 로그인, 회원가입, 게시글 작성처럼 서버에 데이터를 보내 처리하게 할 때 사용한다. POST라고 해서 무조건 안전한 것은 아니고 실제 서비스에서는 HTTPS가 필요하다.

4. form의 action, method, name

form은 사용자가 입력한 데이터를 서버로 보내는 HTML 태그다.
action은 데이터를 보낼 주소, method는 GET/POST 같은 전송 방식 name은 서버가 입력값을 구분할 때 사용하는 key 이름이다.
name은 서버에서 값을 꺼낼 때 쓰는 이름

5. HTML 기본 구조: html, head, body

html은 문서 전체를 감싸는 최상위 태그다.
head에는 제목, 문자 인코딩, CSS 연결 같은 메타 정보가 들어가고 body에는 실제 화면에 보이는 내용이 들어간다.
head는 브라우저와 검색 엔진이 참고하는 정보 영역이고 body는 사용자가 실제로 보는 화면 영역

6. CSS 선택자, 박스 모델

CSS 선택자는 어떤 HTML 요소에 스타일을 적용할지 고르는 문법이다.
박스 모델은 모든 HTML 요소가 content, padding, border, margin으로 이루어진 상자처럼 배치된다는 개념이다.
inline - 나란히 배치됨 줄바꿈 안 됨, width, height 직접 지정이 잘 안 먹음 위아래 margin/padding 조절이 제한적임. 
block - 기본적으로 한 줄 전체 차지, 자동으로 줄바꿈됨, width, height 지정 가능, margin, padding 적용 잘 됨.
inilne-block - 나란히 배치됨, width, height 지정 가능, margin, padding 적용 가능.
