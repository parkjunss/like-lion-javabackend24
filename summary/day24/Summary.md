## 1. 학습 태도 및 학습 방법 점검
- **자체 점검의 중요성:** 강의를 듣는 과정에서 스스로의 기억에 남는 내용을 정리하거나, 궁금한 점을 정의(定義)하는 시간을 가지는 것이 효과적입니다.
- **능동적 학습 자세:** 강의 내용이 어렵거나 이해가 안 되는 부분이 있다면, 막연히 어려워하기보다 적절한 채널을 통해 즉시 질문하고 해결하려는 노력이 필요합니다.
- **지속적인 참여의 습관:** 강의에 꾸준히 참여하여 공백 없이 진도를 따라가는 것이 가장 중요하며, 진도가 많이 지난 후 '하나도 모르겠다'는 상태가 되지 않도록 중간중간 질문하며 학습에 임해야 합니다.

## 2. 웹 구조의 이해: HTML과 CSS의 역할
- **HTML의 역할 (뼈대):** 웹 문서의 구조와 내용을 담는 언어(마크업 언어)입니다. 웹 화면에 보여줄 기본적인 '문서'를 만드는 역할을 합니다.
- **CSS의 역할 (꾸미기):** HTML로 만들어진 내용들이 어떻게 보여질지(색상, 크기, 위치, 모양 등)를 결정하는 스타일시트 언어입니다.
- **두 기술의 목적:**
    - 둘 다 HTML 문서를 꾸미거나(스타일링) 또는 데이터에 값을 넣어 조작하기 위해 존재합니다.
    - **핵심은 ‘접근’:** 자바스크립트를 포함하여, 원하는 HTML 요소(요소)를 정확하게 찾아가고 접근하는 능력(선택자 활용)이 가장 중요합니다.
- **시맨틱 태그 사용:** `<div />` 태그처럼 모든 것을 담을 수 있는 태그 외에도, `header`, `footer`, `nav`, `section` 등처럼 해당 요소가 가지는 '의미(Semantic)'를 담는 태그를 사용하는 것이 좋습니다.

## 3. CSS 심화 학습: 레이아웃과 요소 제어
- **CSS 선택자(Selector)의 활용:**
    - **기본 선택자:** 태그 선택자, `id` 선택자, `class` 선택자 등을 사용하여 원하는 요소를 찾아갑니다.
    - **속성 선택자:** 요소가 가지고 있는 속성(예: `type="text"`)이나 속성의 값(Value)을 이용해 특정 요소를 정교하게 찾을 수 있습니다.
    - **복합 선택자:** 자손 선택자, 자식 선택자, 인접한 형제 선택자 등 다양한 조합을 사용하여 원하는 요소를 세밀하게 지정할 수 있습니다.
- **요소의 배치(Layout) 원리:**
    - **기본 흐름:** 기본적으로 HTML 요소들은 내용물이 흐르듯이 순차적으로 배치됩니다.
    - **Positioning (배치):** 요소의 위치를 자유롭게 지정할 수 있는 `relative`, `absolute`, `fixed` 등의 속성을 사용하여, 원하는 위치(왼쪽, 위 등)에 강제로 배치할 수 있습니다.
    - **Display (표시 방식):**
        - **`block` (블록):** 한 요소를 전체 너비에 걸쳐 배치하고, 다음 요소와는 구분되어 순차적으로 배열됩니다.
        - **`inline` (인라인):** 내용물만큼만 너비를 차지하며, 다른 요소들과 같은 줄에 흐르듯이 배열됩니다.
        - **`flex`:** 복잡한 레이아웃을 쉽게 구현할 수 있도록 도와주는 강력한 배치 방식입니다. (화면 배치 구성의 핵심 도구)
        - **`float`:** 요소를 지면에서 띄워(수평으로) 배치할 때 사용하며, 주변 요소들이 이 상자 밑으로 흐르듯이 배치되게 돕습니다.

## 4. JavaScript의 등장 배경과 핵심 원리
- **JavaScript의 목적:** HTML과 CSS를 넘어, 웹 문서에 동적인 기능(Dynamic Content)을 추가하고 조작하기 위해 탄생한 언어입니다.
- **초기 배경:** 초기 웹 페이지는 내용이 정적(Static)하여 방문자가 반복적으로 이용하기 어려웠기 때문에, 방문자 카운트, 실시간 시간 정보 표시 등 변화하는 요소를 추가할 필요성이 생겼습니다.
- **발전 과정:** 초기에는 CGI 같은 복잡한 기술을 사용했지만, 클라이언트 측에서 가볍고 빠르게 동작하는 도구의 필요성이 대두되었고, 이를 해결하기 위해 JavaScript가 개발되었습니다.
- **기술적 특징:**
    - **브라우저 최적화:** JavaScript는 웹 브라우저 내에서 동작하도록 매우 최적화된 언어입니다.
    - **풀스택 가능성:** 현재는 브라우저(프론트엔드)뿐만 아니라 Node.js 같은 엔진을 통해 브라우저 밖에서도 동작할 수 있어, 백엔드와 프론트엔드를 모두 다룰 수 있는(Full-Stack) 언어로 자리 잡았습니다.
- **공식 명칭과 표준화:**
    - 공식 명칭은 **ECMAScript**입니다.
    - JavaScript는 자바(Java)라는 인기 있는 언어의 이름에서 영감을 얻어 이름이 붙었으나, 기술적으로는 독립적입니다.
    - 현재도 ES5, ES6 등 지속적으로 문법이 표준화되고 발전하고 있습니다.

## 5. JavaScript를 활용한 동적 웹 구현 원리
- **개발자 도구 활용:** 브라우저의 개발자 도구(F12)를 이용해 콘솔 탭에서 JavaScript 코드를 직접 실행하며 테스트할 수 있습니다.
- **DOM(Document Object Model) 접근:**
    - JavaScript가 웹 페이지에서 가장 강력한 기능을 발휘하는 영역입니다. 웹 페이지의 구조(HTML)를 JavaScript가 이해할 수 있는 객체(DOM)로 만듭니다.
    - **요소 선택:** CSS 선택자와 마찬가지로 `document.querySelector()`나 `document.querySelectorAll()` 같은 기능을 이용해 문서 내의 특정 요소에 접근합니다.
    - **요소 조작:** 일단 요소에 접근(선택)하면, 해당 요소의 내용을 변경하거나, 속성(Attribute) 값을 읽고/쓰거나, 아예 새로운 HTML 태그를 삽입하는 등 구조적인 조작(Manipulation)이 가능합니다.
- **핵심 기능:** JavaScript는 단순히 내용을 표시하는 것을 넘어, 웹 페이지에 ‘실행 가능한 논리’를 구현하는 프로그래밍 언어입니다.

## 1. 자바스크립트(JavaScript) 개요 및 활용
- **정의 및 목적:** 자바스크립트는 원래 웹 브라우저에서 동적인 표현(애니메이션 등)을 위해 만들어진 프로그래밍 언어입니다.
- **확장성:** 초기에는 브라우저에 국한되었으나, 현재는 Node.js 등을 통해 브라우저 밖에서도 동작할 수 있습니다.
- **실습 환경:** 코드 샌드박스 등 다양한 도구를 이용해 별도의 설치 없이 코드를 실행해 볼 수 있습니다.

## 2. JS 기본 사용 방법 (HTML/외부 파일 연동)
- **HTML 내 통합:** 자바스크립트를 HTML에서 사용하려면 `<script>` 태그를 사용하고, 그 안에 코드를 작성합니다. (예: `alert('Hello')`)
- **외부 파일 연동:** 코드를 별도의 파일(`*.js`)로 분리하여 관리하는 것이 일반적이며, 이 경우 `<script src="파일이름.js"></script>` 태그와 `src` 속성을 사용해 외부 파일을 가져옵니다.
- **실행 순서:** JS는 코드를 위에서 아래로(상단부터 순차적으로) 해석하고 실행합니다. 따라서 코드의 위치가 실행 순서에 영향을 미칩니다.
- **디버깅 용도:** `console.log()`는 콘솔에 특정 값을 출력해 보는 디버깅 용도로 사용됩니다. 이는 다른 언어의 `System.out.println()`과 유사합니다.

## 3. JS 기본 문법 및 데이터 타입 이해
- **약형 언어(Weakly Typed):** 자바스크립트는 변수가 담을 수 있는 값의 타입이 유연합니다. 변수 하나에 숫자, 문자열, 객체 등 다양한 타입의 값이 들어갈 수 있습니다.
- **자료형 확인:** `typeof` 예약어를 사용하여 변수가 현재 어떤 타입인지 확인할 수 있습니다.
- **템플릿 리터럴:** 백틱(`)을 이용하는 템플릿 리터럴은 변수를 문자열에 삽입하여 포매팅하기 쉽게 만들어주는 ES6 문법입니다. (자바의 `String.format`과 유사)

## 4. 변수 선언, 스코프, 그리고 예약어 비교 (핵심)
자바스크립트의 변수 선언 방식은 세 가지가 있으며, 각각의 특징과 사용 권장 여부가 중요합니다.

### A. `var` (구식 방식 - 사용 지양)
- **특징:**
    *   **재선언 및 재할당 가능:** 변수를 여러 번 선언해도 오류가 발생하지 않습니다.
    *   **호이스팅(Hoisting):** 변수가 실제로 선언된 위치가 아니더라도, JS 엔진이 스크립트 전체를 스캔하면서 해당 변수를 미리 인식합니다. (이로 인해 코드를 예측하기 어렵게 만듭니다.)
    *   **스코프:** 함수 스코프를 따르기 때문에, `{}`로 감싼 블록(if문, for문 등) 내부에서 선언해도 외부에서 접근할 수 있는 문제가 발생할 수 있습니다.
- **문제점:** 위 세 가지 특징 때문에 변수의 범위가 예상치 못하게 넓어지고(Scope Leak), 코드가 복잡해지기 때문에 현재는 사용하지 않는 것이 좋습니다.

### B. `let` (변수 선언)
- **특징:**
    *   **블록 스코프:** `{}`와 같은 블록 단위로 스코프가 한정됩니다. (가장 중요한 특징)
    *   **재할당 가능:** 값을 변경할 수 있습니다.
    *   **호이스팅:** 초기화가 되지 않은 변수에 접근할 경우 `ReferenceError`와 같은 명확한 오류가 발생합니다.
- **용도:** 값이 변경될 수 있는 변수를 선언할 때 사용합니다.

### C. `const` (상수 선언)
- **특징:**
    *   **블록 스코프:** `let`과 마찬가지로 블록 단위로 스코프가 한정됩니다.
    *   **재할당 불가:** 선언 시 값을 지정하면 그 값은 이후 변경할 수 없습니다. (상수 개념)
    *   **호이스팅:** `let`과 동일하게 초기화되지 않은 변수 접근 시 오류가 발생합니다.
- **권장 사항:** 변수 값의 변화가 없을 것으로 예상되는 모든 경우에 `const`를 사용하는 것이 가장 안전한 코딩 습관입니다.

## 5. 고급 문법 및 코드 작성 팁
- **세미콜론(;)의 중요성:**
    *   자바스크립트 문법 자체는 세미콜론이 필수가 아닐 수 있지만, 실제로 코드가 웹 환경(네트워크)을 통해 전송되거나 압축될 때 세미콜론이 누락될 위험이 매우 높습니다.
    *   따라서 잠재적인 오류를 방지하고 안전성을 높이기 위해 세미콜론을 사용하는 것을 **강력히 권장**합니다.
- **문자열 메서드:** 자바스크립트의 문자열(String) 또한 내장된 객체이므로, 자바와 유사하게 다양한 문자열 관련 함수(메서드)들을 사용할 수 있습니다.
- **Unicode 처리:** 자바스크립트는 문자열을 처리할 때 유니코드 문자를 비교적 유연하게 다루며, 다양한 방식으로 문자를 표현할 수 있습니다. (예: 일반 따옴표, 꺾쇠괄호, 쌍따옴표 등 타입 구분에 덜 민감함)

## 1. 자바스크립트 기초 이해: 데이터 타입과 값

*   **타입 시스템 이해:**
    *   자바스크립트는 변수 자체에 타입이 없는 것처럼 보일 수 있지만, 값 자체(Value)에는 `Boolean`, `Number`, `String` 등 명확한 기본 타입이 존재함을 이해해야 합니다.
    *   데이터를 다룰 때는 값이 어떤 타입인지 아는 것이 중요합니다.
*   **`null`과 `undefined`의 구분:**
    *   `undefined`: 변수를 선언만 하고 아직 값을 할당하지 않았을 때의 상태를 의미합니다.
    *   `null`: 의도적으로 "값이 없다"라고 개발자가 명시적으로 지정한 상태를 의미합니다.
    *   **핵심 차이:** 이 둘은 개념적으로 다릅니다.

## 2. 연산자 사용법: 비교 연산자와 논리 연산자

*   **산술 연산자:** 기본적인 덧셈, 뺄셈, 곱셈, 나눗셈, 나머지 연산자(`+`, `-`, `*`, `/`, `%`)는 다른 언어와 동일하게 작동합니다.
*   **비교 연산자 (Comparison Operators):**
    *   **느슨한 비교 (`==`):** 값만 비교합니다. 타입이 달라도 같은 값으로 판단하여 `true`를 반환하는 경향이 있습니다. (예: `"1" == 1`은 `true`로 판단 가능)
    *   **엄격한 비교 (`===`):** 값과 **타입**을 모두 비교합니다. 타입이 다르면 무조건 `false`를 반환합니다.
    *   **권장 사항:** 자바스크립트에서 정확한 판단을 위해 항상 타입까지 확인하는 **엄격 비교 (`===`)** 사용을 권장합니다.
*   **논리 연산자:**
    *   `&&` (AND): 양쪽 조건이 모두 `true`일 때만 `true`를 반환합니다.
    *   `||` (OR): 둘 중 하나라도 `true`일 때 `true`를 반환합니다.
    *   `!` (NOT): 조건을 반대로 뒤집습니다.
*   **학습 조언:** 연산자들은 이론만 보기보다 직접 코드로 입력하고 실행해 보는 실습 과정이 필수적입니다.

## 3. 함수 개념 이해: 함수의 종류와 특징

*   **함수의 이해 (First-Class Citizen):**
    *   자바스크립트에서는 함수(Function) 자체가 하나의 객체(Object)로 취급됩니다.
    *   함수는 다른 함수의 매개변수로 전달되거나, 변수에 할당되거나, 리턴 값으로 나올 수 있습니다. (이를 **1등급 객체** 취급이라고 합니다.)
*   **함수 정의 방법과 특징:**
    *   **1. 함수 선언문 (Function Declaration):** `function add(a, b) { ... }` 형태로 정의합니다.
        *   **특징 1: 호이스팅 (Hoisting):** 함수 이름만으로도 해당 함수가 코드의 어느 위치에서든 호출될 수 있습니다.
        *   **특징 2: 중복 정의 가능:** 같은 이름으로 여러 번 정의해도 에러가 발생하지 않습니다. (가장 마지막에 정의된 내용이 우선하여 사용됨)
    *   **2. 함수 표현식 / 변수 할당:** `const add = function(a, b) { ... }` 형태로 정의합니다.
        *   **특징 1: 범위 제한:** `let`이나 `const` 같은 변수 키워드를 사용하기 때문에, 해당 파일 내에서 중복 정의할 수 없습니다.
        *   **권장 사항:** 변하지 않아야 하는 함수는 `const`를 사용하는 것이 더 안전합니다.
*   **⭐ 함수 전달 시 주의사항 (매우 중요):**
    *   함수를 매개변수로 전달할 때 **괄호(`()`)를 붙이지 않아야 합니다.**
    *   **`func` (괄호 없음):** 함수 자체(객체)를 매개변수로 전달합니다. (정상적인 방법)
    *   **`func()` (괄호 있음):** 함수를 즉시 실행한 **결과 값(리턴 값)**을 매개변수로 전달합니다. (전달하는 것이 아님)

## 4. 심화 주제: 화살표 함수(Arrow Function)와 함수 전달

*   **화살표 함수 문법 (Arrow Function):** `(a, b) => { ... }` 형태로 정의하며, 문법적으로 간결합니다.
    *   **사용법:**
        *   **매개변수 1개:** 괄호 생략 가능. `a => a * 2`
        *   **매개변수 2개 이상:** 괄호 사용 필수. `(a, b) => ...`
        *   **리턴 값:**
            *   단일 표현식의 경우, `{}`나 `return` 키워드 없이 자동으로 리턴됩니다. (`(a, b) => a + b`)
            *   다중 표현식의 경우, 반드시 `return` 키워드를 사용해야 합니다.
*   **함수 스타일의 비교 및 활용:**
    *   자바스크립트 개발자들이 코드를 작성할 때 간결함을 위해 화살표 함수를 많이 사용하므로, 두 가지 형태(일반 함수, 화살표 함수) 모두 익숙하게 다룰 수 있어야 합니다.
    *   **배움의 목표:** 코드의 목적과 기능이 같다는 것을 인지하고, 어떤 문법을 사용하든 헷갈리지 않도록 익숙해지는 것이 가장 중요합니다.

## 1. 자바스크립트의 등장 배경 및 진화 과정
- **필요성 제기:** 초기 웹 페이지(HTML)는 정적인 정보 전달에만 사용되었으며, 사용자가 지속적으로 상호작용하거나 새로운 데이터를 요구하는 서비스(SNS, 유튜브 등)를 구현하기에는 한계가 있었다.
- **자바스크립트의 탄생:** 웹 내에서 단순히 화면을 보여주는 수준을 넘어, 페이지에 동적인 동작과 기능을 추가할 필요성이 생겼고, 이에 따라 자바스크립트가 탄생했다.
- **개발사적 진화:**
    - 초기 자바스크립트는 달력 표시와 같은 간단한 클라이언트 측 작업을 처리하는 용도로만 쓰였다.
    - **AJAX (Asynchronous JavaScript and XML):** 페이지 전체를 새로 로드하는 대신, 배경(비동기적)에서 서버와 데이터를 주고받아 화면의 일부만 동적으로 변경하는 기술을 발표하며 JS의 활용도가 혁신적으로 높아졌다.
    - **Node.js의 등장:** 자바스크립트가 브라우저(클라이언트) 환경을 넘어 서버(백엔드) 환경에서도 구동될 수 있게 되면서 그 위상이 크게 높아졌다.
- **지속적인 표준화:** 초기 자바스크립트는 문법적 일관성이 떨어지고 취약점이 많았기 때문에, ES5, ES6, ES7 등의 표준을 지속적으로 정립하며 언어가 발전하고 있다.

## 2. 자바스크립트의 핵심 개념 및 문법 구조
- **타입 시스템 (Typing):**
    - **자바(Java)와의 차이점:** 자바는 변수 선언 시 타입을 명확하게 정해야 하는 **강한 타입(Static Typing)** 언어인 반면, 자바스크립트는 변수에 숫자, 문자열, 함수, 객체 등 어떤 타입의 값이라도 담을 수 있는 **약한 타입(Weak Typing)** 언어이다.
    - **타입 보완:** 타입의 불명확성이라는 단점을 보완하기 위해 **TypeScript**가 등장했다. TypeScript는 자바스크립트의 상위 집합으로, 정적 타입 체크 기능을 추가하여 안정성을 높인 언어이다.
- **변수 선언 및 스코프:**
    - **`var`:** 가장 오래된 선언 방식이다. 스코프(Scope)가 함수 단위로 적용되며, 호이스팅(Hoisting) 등의 특성 때문에 코드의 가독성과 안정성이 떨어진다.
    - **`let`과 `const` (권장):**
        - **`let`:** 재할당이 가능한 변수 선언 방식이다.
        - **`const`:** 재할당이 불가능한 상수 선언 방식이다.
        - **권장 이유:** `let`과 `const`는 블록 스코프(Block Scope)를 지원하며, `var`에서 발생할 수 있는 스코프 문제와 오류를 방지하므로 사용을 권장한다.
    - **상수 활용:** DOM(문서 객체 모델)에서 요소를 가져오는 등 객체 참조를 사용할 때는 내부 값은 변할 수 있어도 객체 자체는 바뀌지 않는 경우가 많으므로, **`const`**를 사용하는 것이 일반적이다.
- **선언어와 개념:**
    - **호이스팅(Hoisting):** 자바스크립트는 인터프리터 언어임에도 불구하고, 코드가 실행되기 전에 전체 문서를 스캔하여 변수나 함수 등의 선언 정보를 미리 인식하고 사용하는 특성(선언보다 먼저 사용 가능)을 가진다.
    - **비교 연산자:**
        - `==` (느슨한 비교): 타입이 다름에도 불구하고 자바스크립트가 임의로 형변환을 시도하여 비교를 수행한다.
        - `===` (엄격한 비교): 타입과 값이 모두 같아야 `true`를 반환하는 가장 정확한 비교 방식이므로 이를 사용하는 것이 필수적이다.
- **`null`과 `undefined`:**
    - **`undefined`:** 값이 초기화되지 않아 시스템적으로 할당된 상태를 의미한다.
    - **`null`:** 개발자가 **의도적으로** 값이 없음을 명시적으로 할당한 값이다.

## 3. 자바스크립트의 주요 특징 및 활용 방법
- **함수 (Functions):**

    - **문법 구조:** `function` 키워드를 사용해 선언하며, 매개변수와 리턴 타입 등을 정의한다.
    - **화살표 함수 (Arrow Functions):** `=>`를 사용해 함수를 정의하는 간결한 문법이다. 매개변수가 하나일 때 괄호 생략, 본문이 한 줄일 때 `return` 키워드 및 중괄호 생략이 가능하다.

    - **오버로딩(Overloading) 개념 없음:** 다른 언어에서처럼 함수 이름만 같게 여러 매개변수를 받는 함수를 정의할 수 없다.
    - **대체 메커니즘:** 대신, 자바스크립트는 함수가 실행될 때 전달된 모든 인자들을 담는 배열 같은 `arguments` 객체를 자동으로 제공한다. 이 객체를 통해 받은 인자의 개수와 값에 접근할 수 있다.
- **활용 목적: DOM 조작:**
    - 자바스크립트는 브라우저 내에서 동작하는 언어로서, HTML 요소(DOM)를 조작하는 것이 핵심 역할을 한다.
    - **기능:** HTML 요소를 찾고, 스타일을 적용하거나(CSS 조작), 요소의 내용을 교체하거나, 요소의 값을 얻어내는 등의 작업을 수행하는 데 사용된다.
- **함수의 전달 및 반환:**
    - 함수 자체를 매개변수로 전달하거나(함수 자체를 인수로 받음), 함수의 결과를 또 다른 함수로 반환하는 것 모두 가능한 것이 자바스크립트의 강력한 특징이다. (예: `process()` 함수를 인수로 전달하여 실행)

- **함수의 특성:** 자바스크립트에서 함수는 **일급 백터(First-Class Citizen)**로 취급됩니다.
- **일급 백터가 가능하게 하는 것:**
    - 변수에 함수를 할당(저장)할 수 있습니다.
    - 함수를 다른 함수에 인자로 전달(콜백)할 수 있습니다.
    - 함수의 리턴 값(반환값)으로 다른 함수를 반환할 수 있습니다.
- **OOP 관점:** 자바스크립트는 객체 지향 언어로, '개체(Object)'라는 개념을 기반으로 합니다. (자바와 유사하지만, 클래스 기반이 아닌 방식으로 객체 생성 및 처리가 가능함)
- **메서드 체이닝:** 함수의 실행 결과 값이 또 다른 함수(혹은 객체)가 되어, 그 결과를 가지고 다시 메서드를 실행시키는 연속적인 처리가 가능합니다.

## 2. 자바스크립트 객체 생성 및 활용
- **객체 생성:**
    - JS는 `new` 키워드와 같은 명시적인 방식 없이도, `{}` 중괄호 사용 등을 통해 바로 객체를 생성할 수 있습니다.
    - 객체는 키(Key)와 값(Value)의 쌍으로 구성되며, 변수명 사용이 가능합니다.
- **객체 속성 접근 및 수정:**
    - 점 표기법(예: `object.property`)을 사용하여 객체 내부의 값(속성)을 읽거나 변경할 수 있습니다.
    - **속성 추가:** 객체가 정의될 때 없었던 속성이라도 동적으로 추가할 수 있습니다.
- **객체 메서드 추가:**
    - 객체 생성 시 필드(속성) 외에 함수(기능)를 추가하여 사용할 수 있습니다.
    - 이 추가된 함수는 변수처럼 취급되어 객체의 기능으로 사용됩니다.

## 3. 데이터 교환 표준: XML과 JSON의 비교
- **데이터 교환의 필요성:** 클라이언트와 서버 등 시스템 간에 데이터를 주고받기 위해서는 구조화된 '약속'이 필요합니다.
- **XML (eXtensible Markup Language):**
    - **특징:** 항상 정해진 문서 규격(Root 태그, 태그 구조 등)을 가집니다.
    - **구조:** 태그(Tag)를 사용하여 데이터의 종류와 구조를 명시적으로 구분합니다. (예: `<name>강경미</name>`)
    - **장점:** 데이터의 구조적 안정성이 높고, 유효성 검사(DTD, 스키마)가 가능하여 신뢰도가 높습니다.
    - **단점:** 데이터의 내용이 텍스트 기반이어서 오버헤드가 크고, 필요한 데이터가 아닌 부가적인 정보(배보다 배꼽처럼)가 많이 포함될 수 있어 복잡합니다.
- **JSON (JavaScript Object Notation):**
    - **도입 배경:** 웹 환경에서 데이터를 주고받을 때 XML의 복잡성과 과도한 데이터 용량 문제가 발생하여, 더 간단하고 효율적인 방식이 필요해졌습니다.
    - **특징:** 자바스크립트 객체 구조를 기반으로 데이터를 표현합니다. (Key와 Value 쌍)
    - **구조:** `{ "키": "값" }` 형태가 기본이며, 배열(`[]`)을 포함할 수 있습니다.
    - **장점:** XML보다 간결하며, 데이터가 자바스크립트 객체 구조와 일치하여 처리와 이용이 매우 쉽습니다.
    - **결론:** 현재 웹 어플리케이션에서 데이터를 주고받는 **가장 많이 사용되고 표준적인 방식**입니다.

## 4. 고급 자바스크립트 개념: `this` 키워드의 이해
- **`this`의 의미:** 자바스크립트에서 `this`는 현재 코드가 실행되는 '맥락(Context)'을 가리키는 키워드입니다. 즉, '나'를 의미합니다.
- **함수형 `this` (Function Call):**
    - 일반 함수 내에서 사용되는 `this`는 **함수가 호출된 객체**를 가리킵니다. (예: `dog.say()`를 실행하면, `this`는 `dog` 객체 자체입니다.)
- **화살표 함수형 `this` (Arrow Function):**
    - 화살표 함수 (`=>`)는 자체적인 `this`를 가지지 않고, **코드가 선언되거나 실행되는 환경의 상위 범위**의 `this`를 그대로 상속(Lexical Scoping)합니다.
    - **차이점:** 일반 함수를 사용하면 `this`가 호출 객체로 바뀌지만, 화살표 함수를 사용하면 호출 객체에 상관없이 외부 상위 객체(`Window` 객체 등)를 참조하게 됩니다.
- **주의 사항:**
    - 객체 메서드 내에서 `this`가 중요한 역할을 하므로, 메서드를 정의할 때는 화살표 함수보다는 일반 함수를 사용하는 것이 바람직합니다.

## 1. 자바스크립트 객체의 특징 및 함수 활용
- **유연한 객체 생성:** 자바스크립트 객체는 반드시 클래스(틀)를 통해 생성될 필요가 없습니다. 틀 없이도 객체를 만들 수 있으며, 생성 후에도 필요에 따라 속성을 추가, 수정하는 것이 자유롭습니다.
- **객체 내부의 행위 (메서드):** 객체는 속성(데이터)뿐만 아니라 행위(함수, 메서드)도 가질 수 있습니다.
- **함수와 1급 객체:** 함수는 자바스크립트에서 '1급 객체'로 취급되므로, 변수처럼 선언하고 값처럼 객체에 속성으로 할당할 수 있습니다.
- **함수 정의 문법:**
    - `객체명.속성명: function () {}` 형태로 사용할 수 있습니다.
    - 함수 자체를 이름으로 사용하거나 (일반 함수 방식), 간결하게 함수명만으로 사용할 수 있습니다.
    - 화살표 함수(`=>`)도 사용할 수 있으나, **`this` 바인딩에 유의해야 합니다.**
- **`this` 주의사항 (중요):**
    - **일반 함수:** 객체 내부에서 함수를 사용할 때 `this`는 해당 함수를 감싸고 있는 객체를 가리킵니다.
    - **화살표 함수:** 객체 내부에서 사용하더라도 `this`는 객체가 아닌 **전역 환경**을 인지합니다. 따라서 객체 내부의 컨텍스트가 중요할 때는 화살표 함수 사용에 주의가 필요합니다.

## 2. 객체 구조 분해 할당 (Object Destructuring)
- **개념 및 목적:** 객체 안에 담겨있는 여러 속성들 중 필요한 값만 추출하여 상수로 선언할 수 있는 문법입니다. 코드를 간결하게 만들고 가독성을 높이는 데 유용합니다.
- **사용법:**
    - 상수를 선언할 때 `{}` 중괄호 안에 해당 객체의 필드 이름(속성명)을 명시합니다.
    - 예: `const { userName, email } = user;`
- **핵심 규칙 및 주의사항:**
    - **필드명 일치:** 구조 분해 할당을 할 때는 객체에 존재하는 **실제 필드명**과 변수명(할당받을 이름)이 일치해야 합니다.
    - **순서 무관:** 구조 분해 시 필드를 꺼내는 순서는 중요하지 않습니다.
    - **불일치 시 처리:** 만약 객체에 존재하지 않는 필드명을 지정하면, 해당 변수는 `undefined` 값을 가지게 됩니다.
- **응용 분야:**
    - **DOM 조작 및 프레임워크:** HTML 요소를 다루는 `Document Object Model` (DOM)처럼 하나의 복잡한 객체가 넘어올 때, 원하는 속성(예: `type`, `id`, `value`)만 구조 분해를 이용해 효율적으로 추출할 수 있습니다.

## 3. 객체 접근 제어와 Getter/Setter 패턴
- **필요성 (캡슐화의 개념):**
    - 자바스크립트에는 자바와 같은 명시적인 `private` 접근 제한자가 없습니다.
    - 하지만 객체의 속성 값이 외부에서 함부로 접근하거나 변경되는 것을 방지하고, 특정 로직을 거쳐 값을 읽거나 쓸 수 있도록 **제어**하는 것이 바람직합니다.
- **Getter/Setter 메서드 활용:**
    - 객체의 속성(필드)에 직접 접근하는 대신, `getter`나 `setter`와 같은 **메서드**를 통해서만 값을 얻거나 변경하게 만드는 것이 이상적입니다.
    - 이 패턴을 통해 **데이터의 유효성 검사(Validation)** 같은 로직을 추가할 수 있습니다. (예: Setter에서 "이름은 두 글자 이상이어야 한다"는 조건을 걸 수 있습니다.)
- **주체(Source of Truth)의 명확화:**
    - Getter/Setter를 사용한다는 것은, 값을 읽거나 쓰는 **주체가 외부가 아닌 메서드(코드) 자체**임을 의미합니다. 이는 단순한 값 전달을 넘어, 로직을 통한 통제권을 부여합니다.
- **결론:** Getter와 Setter는 객체의 내부 상태를 보호하고, 상태 변경 시 필요한 검증 및 부가 로직(비즈니스 로직)을 삽입하기 위한 설계 패턴입니다.
