## 1. JavaScript의 기본적인 특징 및 역사적 이해
*   **JS의 정의 및 동작 환경:**
    *   JavaScript는 원래 브라우저 환경에서 프로그래밍을 동작시키는 언어로 시작했습니다.
    *   현재는 Node.js와 같은 엔진을 통해 브라우저 외부에서도 동작할 수 있습니다.
*   **약형 언어 (Weakly Typed Language):**
    *   Java와 달리 JavaScript는 변수의 자료형(Type)이 명확하게 정해져 있지 않은 약형 언어입니다.
    *   따라서 변수 선언 시 타입을 지정할 필요가 없으며, 이후 다른 타입의 값으로도 재할당이 가능합니다.
*   **버전의 흐름과 권장사항:**
    *   JavaScript는 ES5, ES6 등으로 문법이 끊임없이 업데이트되면서 발전해 왔습니다.
    *   과거의 예약어(`var`)는 여전히 레거시 코드에서 사용되지만, 새로운 코드를 작성할 때는 `let`과 `const` 사용이 권장됩니다.

## 2. JavaScript의 핵심 문법 요소
*   **변수 선언 및 스코프 (Scope):**
    *   **`var`:** JavaScript 초기 버전부터 사용된 예약어입니다.
    *   **호이스팅 (Hoisting):** JavaScript가 인터프리터 방식임에도 불구하고, 변수 선언이 실제 사용 코드보다 위에서 먼저 인식되어 마치 선언된 것처럼 동작하는 현상입니다. (함수도 호이스팅의 영향을 받음)
    *   **스코프 (Scope):** `var`는 함수 단위의 스코프를 가지지만, `let`과 `const`는 블록(Block) 단위 스코프를 가지는 것이 특징입니다.
*   **선언 방식의 차이점:**
    *   **`var`:** 함수 단위의 스코프를 가지며, 변수 중복 선언이 가능합니다. (초기 버전의 특징)
    *   **`let` / `const`:** 블록 단위 스코프를 가지며, 같은 스코프 내에서 변수 중복 선언이 불가능합니다. (현대적인 사용 권장)

## 3. 함수와 함수형 프로그래밍 개념
*   **일급 객체 (First-Class Object):**
    *   JavaScript에서 함수는 단순한 실행 코드가 아니라, 일반 값(Object)과 동일하게 취급됩니다.
    *   따라서 함수를 변수에 할당하거나(Assignment), 함수의 인자(Argument)로 전달하거나(Passing), 함수의 반환 값(Return Value)으로 반환할 수 있습니다.
*   **함수 호출 시 유의점 (매우 중요):**
    *   **함수 자체를 넘길 때 (콜백 함수):** `함수명` (괄호 없음)을 사용합니다. 함수 객체 자체를 인자로 전달합니다.
    *   **함수를 실행하고 결과를 넘길 때:** `함수명()` (괄호 있음)을 사용합니다. 함수가 실행된 '결과 값'이 인자로 전달됩니다.

## 4. 최신 JavaScript 문법 및 고급 활용 (ES6+)
*   **템플릿 리터럴 (Template Literals):**
    *   따옴표 대신 **백틱(Backtick, ``)**을 사용하여 문자열을 정의할 수 있습니다.
    *   이를 통해 문자열 내에서 변수 값이나 표현식(`$${변수}` 또는 `${표현식}`)을 쉽게 포함하여 문자열을 구성할 수 있습니다.
*   **객체 구조 분해 할당 (Object Destructuring):**
    *   객체나 배열에서 필요한 값만 추출하여 변수에 할당하는 편리한 문법입니다.
    *   **활용 예시:**
        *   객체에서 필요한 특정 속성(예: `name`, `age`)만 바로 추출하여 변수로 사용할 수 있습니다.
        *   함수의 매개변수로 객체를 받을 때, 필요한 속성만 바로 구조 분해하여 사용할 수 있습니다.
    *   **주의사항:** 구조 분해 할당 시, 객체의 필드 이름과 변수 이름이 **일치**해야 합니다.
*   **`this` 바인딩과 클래스:**
    *   `this`는 코드가 실행되는 환경이나 문법에 따라 '나 자신'을 가리키는 참조 값이 동적으로 변합니다.
    *   **일반 함수:** `this`가 호출 환경(예: 브라우저의 `window` 객체)을 따라가며 바인딩됩니다.
    *   **화살표 함수 (Arrow Function):** 자신이 정의된 가장 가까운 스코프(Lexical Scope)를 바라보며 `this`를 결정합니다. 따라서 이 스코프가 유지되는 장점이 있습니다.

## 5. 객체 지향 프로그래밍 심화 개념 (OOP Concepts)
*   **Getters와 Setters의 필요성:**
    *   Java와 마찬가지로, 자바스크립트에서도 데이터의 무분별한 직접 접근 및 수정을 막기 위해 Getters와 Setters 개념을 사용합니다.
    *   이는 외부에서 객체의 필드(속성)를 보호하고, 값을 읽거나 쓸 때 필요한 로직(예: 유효성 검사)을 추가하기 위함입니다.
*   **JavaScript에서의 구현:**
    *   JavaScript는 메서드 정의를 통해 속성 접근 시 `get`과 `set`을 사용하여 게터/세터의 기능을 구현할 수 있습니다.
    *   사용할 때 문법적으로는 메서드를 호출하는 것처럼(`user.getName()`) 사용하지만, 실제 동작은 마치 일반 속성에 직접 접근하는 것처럼 보입니다.

## 1. JavaScript 핵심 문법 복습 (Getter/Setter, `this`)
- **JavaScript의 특징:** 자바스크립트는 접근 제한자(Access Modifier)와 같은 엄격한 문법적 구조를 갖고 있지 않습니다.
- **Getter와 Setter:** 이러한 한계를 보완하기 위해 **Getter**와 **Setter**를 사용하여 속성 접근을 제어할 수 있도록 제공합니다.
- **`this`의 이해:** `this` 키워드의 값은 일반 함수와 화살표 함수에서 다르게 인식되므로, 이 점을 기억하는 것이 중요합니다.

## 2. JavaScript 배열 기본 개념 및 생성
- **배열의 본질:** 배열은 단순히 값을 순서대로 저장하는 컨테이너를 넘어, 그 자체로 하나의 **Object**로 취급됩니다.
- **값의 유연성:** 배열 안에는 같은 타입의 값뿐만 아니라, 원시 타입(Primitive)과 객체(Object), 함수 등 서로 다른 타입의 값을 자유롭게 담을 수 있습니다.
- **접근 및 속성:** 배열의 값은 인덱스(예: `arr[0]`, `arr[1]`)를 사용해 접근하며, 배열의 길이는 `.length` 속성으로 확인할 수 있습니다.
- **배열 생성 방식의 차이 (핵심):**
    - **`[]` 사용:** 가장 직관적인 방법으로 값을 담아 배열을 생성합니다.
    - **`new Array()` 사용:** 생성자의 인자 개수에 따라 동작 방식이 달라집니다.
        - **인자가 하나만 들어올 경우:** 해당 인자 값이 아니라, 그 인자 크기(길이)만큼의 비어 있는 공간을 가진 배열이 생성됩니다. (예: `new Array(10)`은 10개의 빈 공간을 가짐)
        - **인자가 두 개 이상 들어올 경우:** 인자로 받은 값들로 구성된 배열이 생성됩니다.
- **배열 할당 및 구조분해:**
    - 배열에 값을 직접 할당할 수 있습니다 (예: `arr[2] = 값`).
    - **배열 구조분해 할당:** 객체 구조분해 할당과 달리 키(Key)의 개념 없이 **순서**가 중요합니다. 배열의 순서대로 값을 변수에 할당받을 수 있습니다.

## 3. 배열 조작 메서드 및 구문 (Mutability와 Spread 연산자)
- **주요 조작 메서드:**
    - **`push()`:** 배열의 **끝(End)**에 값을 추가합니다.
    - **`pop()`:** 배열의 **끝**에 있는 값을 제거하고 반환합니다.
    - **`unshift()`:** 배열의 **처음(Start)**에 값을 추가합니다.
    - **`shift()`:** 배열의 **처음**에 있는 값을 제거하고 반환합니다.
- **내장 함수 (Built-in Methods):**
    - **`concat()`:** 두 개 이상의 배열을 합쳐서 완전히 **새로운 배열**을 생성합니다.
    - **`slice()`:** 지정된 인덱스 범위의 요소를 잘라내어 **새로운 배열**을 가져옵니다.
- **Spread 연산자 (`...`) 활용:**
    - **배열 전개 (Spreading):** 배열의 내용을 다른 곳(새로운 배열, 함수 호출 등)으로 '펼쳐서' 복사하는 데 사용됩니다.
    - **활용처:** 배열 복사나 다른 배열과의 병합 시 유용하며, 불필요한 복사 과정 없이 간편하게 배열을 생성할 수 있습니다.

## 4. 배열의 반복 및 고차 함수 활용 (Functional Programming)
- **반복문 (Iteration):**
    - 배열의 요소를 순회할 때, 전통적인 `for` 루프나 `for...of`와 같은 반복문을 사용할 수 있습니다.
    - **강점:** 자바스크립트의 반복문 구문들이 배열에서도 유사하게 작동하여 범용성이 높습니다.
- **고차 함수 (Higher-Order Functions):**
    - **`forEach()`:** 배열의 각 요소에 대해 특정 함수를 실행(Side Effect)시키는 데 사용됩니다. (반복문을 돌면서 뭔가를 '하겠다'에 초점).
    - **`map()`:** 배열의 각 요소를 함수에 통과시키고, **새로운 결과 값들로 이루어진 새로운 배열을 반환**합니다. (원본 배열은 변경되지 않습니다).
    - **`filter()`:** 배열의 요소들을 순회하며, **특정 조건을 만족하는 요소들만 걸러내어 새로운 배열을 반환**합니다.
        - **필수 조건:** `filter()`에 전달하는 함수는 반드시 **`true` 또는 `false`** 값을 반환해야 합니다.

***
**💡 학습 요약:**
자바스크립트의 배열은 단순한 자료구조가 아니라 다양한 내장 함수와 메서드를 제공하는 객체이며, `map`, `filter`와 같은 고차 함수들을 통해 코드를 더 간결하고 선언적으로 처리할 수 있습니다. 이 함수들은 모두 **새로운 배열(불변성)**을 생성하는 방식으로 작동합니다.

## 1. 배열의 고차 함수(High-Order Functions) 활용
배열 객체가 제공하는 강력한 내장 메서드들은 반복문 작성의 번거로움을 줄이고 코드를 간결하게 만듭니다.

- **`forEach` (포어이치):**
    - 배열의 모든 요소에 대해 지정된 함수를 순차적으로 실행합니다.
    - **주요 역할:** 각 요소를 순회하며 특정 작업을 수행할 때 사용합니다. (값을 누적하거나, 사이드 이펙트를 일으키는 작업에 주로 사용)
- **`map` (맵):**
    - 배열의 각 요소를 변환하여 새로운 배열을 생성합니다.
    - **주요 역할:** 기존 배열의 형태와 길이는 유지하되, 요소의 값만 새로운 규칙에 따라 바꿀 때(변환) 사용합니다.
- **`filter` (필터):**
    - 배열의 요소들을 조건을 만족하는지 확인하여, 조건을 통과한 요소들로만 이루어진 새로운 배열을 생성합니다.
    - **주요 역할:** 특정 기준(예: 짝수만)에 맞는 데이터만 걸러낼 때 사용합니다.
- **`reduce` (리듀스):**
    - 배열의 요소들을 하나의 값으로 '누적(Accumulation)'하여 최종 결과값을 도출합니다.
    - **작동 방식:** 초기값을 설정하고, 배열의 각 요소가 순회할 때마다 누적된 값과 현재 요소를 연산하여 다음 누적 값으로 덮어씁니다.
    - **활용:** 합계 계산, 평균 계산 등 여러 요소를 하나의 값으로 종합할 때 유용합니다.
    - **주의:** 리듀스는 내부적으로 재귀함수처럼 동작하므로, 같은 로직을 재귀로 구현하는 것보다 효율적이지만, 과도한 사용은 성능 저하를 유발할 수 있습니다.

## 2. 데이터 순회 및 반복문 이해 (Loops)
자바스크립트에서는 데이터를 반복적으로 처리할 때 다양한 반복문 구조와 배열/객체 순회 문법을 제공합니다.

- **전통적인 반복문 (`for`와 `while`):**
    - 자바나 자바스크립트의 일반적인 `for`나 `while` 문은 기본적인 인덱스 기반 순회에 사용됩니다.
- **개선된 순회 문법:**
    - **`for...of`:** 배열(Array)과 같이 반복 가능한(Iterable) 객체의 **요소 값**을 하나씩 순회하며 접근할 때 사용합니다. (가장 일반적인 배열 순회에 권장됨)
    - **`for...in`:** 객체(Object)의 **속성 키(Key)**를 순차적으로 가져올 때 사용합니다. (객체의 속성이나 필드를 순회할 때 적합)
- **반복 제어문:**
    - **`break`:** 반복문을 즉시 종료하고 빠져나갈 때 사용합니다.
    - **`continue`:** 현재 반복에서 다음 조건으로 건너뛰어(skip) 다음 요소를 처리할 때 사용합니다.

## 3. 개발 학습 방법 및 심화 학습 조언
코드의 작동 원리를 이해하는 것과 실제 코드를 작성하는 것은 다릅니다. 실력 향상을 위해서는 능동적이고 깊이 있는 학습 태도가 중요합니다.

- **개념 이해의 심화:**
    - 반복문(Loop)을 통해 수행하는 모든 작업은 결국 '순회(Iteration)' 과정입니다. 고차 함수(`map`, `filter` 등)를 사용하는 이유는, 이러한 순회 작업을 직접 반복문으로 작성하는 것보다 **훨씬 간편하고 가독성이 높기 때문**입니다.
    - 코드를 처음 접할 때, 무조건 동작하는 결과만 보고 넘어가지 말고, **이 코드가 내부적으로 어떤 원리(예: `reduce`가 어떻게 누적하는지)로 작동하는지** 과정을 분해하며 이해하려는 노력이 중요합니다.
- **실습 및 자율 학습의 중요성:**
    - **변수 변경 및 예측:** 코드를 단순히 복사/실행하는 것보다, 변수 값이나 입력 값을 직접 바꿔가면서(테스트 케이스 변경) 실행 결과를 예측해 보는 훈련이 필요합니다.
    - **오류 경험:** 오류가 발생하는 경험을 하는 것이 학습에 더 효과적입니다. 오류를 만나야 '무엇이 잘못되었는지'를 능동적으로 생각하게 되기 때문입니다.
    - **자료 검색 활용:** 개념적으로 이해가 안 되는 부분은 MDN(Mozilla Developer Network)과 같은 공식 문서나 웹 스쿨을 통해 반드시 찾아보고 실습하는 것이 도움이 됩니다.
- **학습 마인드셋:**
    - 프로그래밍 실력은 단기간의 수업 몇 시간으로 습득되는 것이 아닙니다. 꾸준히, 많이 문제를 풀고, 오류를 경험하며, 끊임없이 자신의 코드를 발전시켜 나가는 과정 자체가 가장 중요합니다.

## 1. JavaScript 배열과 객체 활용 심화 (Array Methods)

**◼ JS와 Java 배열의 차이점**
*   **Java 배열:** 선언 시점부터 담을 데이터 타입과 크기가 고정됩니다.
*   **JavaScript 배열:** 언어 자체의 특징상 타입을 명시하지 않아 다양한 타입의 데이터를 담을 수 있는 유연한 배열이 만들어집니다.
*   **접근 및 조작:** 배열의 값을 넣거나 빼는 것은 인덱스(index)를 사용합니다.
*   **객체 활용:** JS는 `Array`와 같은 내장 객체를 제공하며, `[]`와 같이 직관적으로 생성하는 방식이 `new Array()` 문법보다 효율적입니다. (다만, `new` 문법을 사용하는 것도 가능합니다.)

**◼ 배열의 핵심 기능 (Array 객체의 메소드)**
*   **기본 메소드:** `push`, `shift`, `pop`, `unshift` 등의 메소드를 이용해 배열의 요소를 추가/삭제할 수 있습니다.
*   **순회 및 처리 함수:** `Array.prototype`에서 제공하는 함수들이 강력합니다.
    *   **`forEach()`:** 반복문을 돌면서 각 요소를 순회하며 작업을 수행할 때 사용합니다.
    *   **`map()`:** 각 요소를 기반으로 새로운 값에 매핑하여 *새로운 배열을 생성*하고 반환할 때 사용합니다. (데이터 변형 및 새 배열 생성 시 유용)
    *   **`filter()`:** 조건식(Condition)을 사용하여 해당 조건이 `true`인 요소들만 골라내어 *새로운 배열을 생성*할 때 사용합니다.
    *   **`reduce()`:** 배열의 모든 요소를 순회하며 *누적 변수(accumulator)*에 값을 계속 담아 연산 결과를 하나의 값으로 요약할 때 사용합니다.

## 2. 데이터 전송과 객체 매핑 이해 (Client-Server Communication)

**◼ 백엔드(Backend)와 프론트엔드(Frontend) 간의 데이터 흐름**
*   백엔드에서 정제된 데이터를 프론트엔드(JS)로 전달할 때, 객체 형태로 직접 주고받을 수 없습니다.
*   **표준화된 형식:** 네트워크를 통해 주고받는 모든 데이터는 반드시 **JSON (JavaScript Object Notation)** 형태의 문자열로 변환됩니다.
*   **JS의 역할:** 자바스크립트는 이 JSON 형태의 문자열을 받아들여 네이티브 자바스크립트 객체 형태로 완전히 매핑(mapping)하여 사용하게 됩니다.
*   **핵심 이해:** 웹 개발에서는 데이터를 객체 형태로 가공하고, 이를 JSON 문자열로 전송하며, 수신 측에서 다시 자바스크립트 객체로 인식하여 사용하는 과정이 빈번하게 일어납니다.

## 3. 개발자로서의 지식 공유 및 성장의 자세

**◼ 지식 공유와 오픈 소스의 가치**
*   개인이 가진 지식이나 노하우를 아낌없이 공개하는 문화(Open Source)는 개발자에게 큰 자산이 됩니다.
*   지식을 공개하는 행위는 단순한 정보 나열을 넘어, 생각의 흐름, 문제를 해결하는 과정(예: "이 코드에서 문제가 발생했고, 나는 이렇게 생각하여 해결했다")을 같이 정리할 때 의미가 있습니다.
*   **경고:** 단순히 수업 내용 요약이나 인공지능(AI)을 활용해 생성된, 본인의 생각이 담기지 않은 정보는 아무런 가치가 없습니다.
*   **가장 중요한 것:** 자신의 *생각*과 *고민의 과정*을 기록하고 정리하는 것이 가장 중요합니다.

**◼ 프로그래밍 사고방식의 함정 (Anti-Pattern)**
*   인간은 오랜 시간 습득한 고정관념(관념)을 가지고 생각하기 때문에, 코드를 짜거나 문제를 해결할 때도 과거의 관념을 바탕으로 오류를 범하는 경우가 많습니다.
*   이러한 고정적인 사고방식이 개발 과정에서 오류를 일으킬 수 있으며, 이를 **안티패턴(Anti-Pattern)**이라고 합니다.
*   개인이 이러한 실수 사례나 개선점을 스스로 정리하고 기록해두면, 나중에 유사한 문제가 발생했을 때 빠르고 정확하게 해결할 수 있습니다.

## 4. JavaScript 객체와 프로토타입 이해 (OOP 개념)

**◼ JS의 객체 정의와 생성 (Class/Function)**
*   JS는 함수를 객체처럼 사용하여 개념적으로 객체(Object)를 정의할 수 있습니다.
*   **`new` 키워드:** 생성자 함수를 사용해 객체를 생성하고, 입력받은 값들을 멤버 변수(`this.변수`)에 할당할 수 있습니다.
*   **ES6 Class:** ES6에서 `class` 문법을 도입하여 객체 정의가 자바와 유사하게 보이도록 문법을 추가했습니다. 그러나 내부적으로 동작 원리는 여전히 **프로토타입 기반**입니다.

**◼ 프로토타입(Prototype)의 개념 (메모리 공유)**
*   **Java와의 차이:** 자바는 객체가 생성될 때마다 속성(멤버 변수)과 메서드 공간을 모두 독립적으로 가지는 경향이 있습니다.
*   **JS의 특징 (공유):** JavaScript는 메모리 효율을 위해 메서드 같은 공통 기능을 **프로토타입이라는 공유 영역**에 저장합니다.
*   **원리:**
    1.  객체를 생성할 때, 각 인스턴스는 고유한 속성(필드)을 가집니다.
    2.  하지만 `say()` 같은 공통 기능(메서드)은 모든 인스턴스가 공유하는 프로토타입 영역을 참조합니다.
    3.  이 덕분에 모든 객체가 동일한 기능(메서드)을 참조하여 사용하게 됩니다.
*   **장점:** 이 공유 영역(`Prototype`)의 기능을 수정하면, 해당 기능을 사용하는 모든 객체(인스턴스)에 한 번에 변화를 줄 수 있습니다.

**◼ 프로토타입 상속**
*   프로토타입은 상속 개념을 구현하는 핵심입니다.
*   `A.call()`과 같은 문법을 이용하거나, `A.prototype`을 사용하여 특정 클래스(부모)의 기능을 자식 클래스에서 가져와 사용할 수 있습니다. 이는 객체 지향의 핵심인 상속 메커니즘을 구현합니다.
*   **결론:** 프로토타입은 객체가 생성될 때 필요한 공통 기능과 속성을 공유하는 **공유 영역**이라고 이해하는 것이 가장 명확합니다.

## 1. JavaScript의 객체 모델: 프로토타입과 클래스
- **기본 구조:** 자바스크립트는 기본적으로 프로토타입(Prototype) 형태의 객체를 이용하며, 이는 자바스크립트 객체 생성의 핵심 원리입니다.
- **프로토타입의 역할:** 객체를 정의할 때, 필드나 메서드를 모든 인스턴스에 복사해 넣지 않고 공유하는 영역으로 프로토타입을 사용합니다.
- **효율성:** 이 공유되는 프로토타입 영역 덕분에, 여러 인스턴스가 동일한 필드나 메서드를 효율적으로 사용하며 메모리 사용량을 절약할 수 있습니다.

## 2. ES6 클래스 문법 이해하기 및 현대적 사용 권장 사항
- **ES6의 도입 배경:** 자바스크립트의 문법 일관성 문제를 개선하기 위해 ES6(ECMAScript 2015)에서 `class`라는 새로운 예약어를 도입했습니다.
- **클래스의 이해:** `class` 문법은 개발자들이 익숙한 클래스 기반 언어(예: Java)의 문법처럼 보이게 만든 '문법적 설탕(Syntactic Sugar)'에 불과합니다.
    - 실제 동작 원리: 내부적으로는 여전히 **프로토타입 기반**으로 작동합니다.
    - 특징: `constructor`와 `method`를 정의하는 과정에서 프로토타입에 메서드가 등록되는 방식입니다.
- **현대적 사용 권장 사항:**
    - 비록 `class` 문법을 사용할 수 있게 되었지만, 현재 자바스크립트 생태계(특히 리액트 등)에서는 **클래스 기반보다는 함수(Function) 기반의 문법**을 사용하는 것이 더 일반적이며 권장됩니다.
    - 학습 시, 클래스 문법 자체에 너무 깊이 매몰되기보다는 **함수문법을 활용하는 방식**을 이해하는 것이 실무에 더 중요합니다.

## 3. 자바스크립트 예외 처리 (Error Handling)
- **목적:** 프로그램이 동작하는 도중에 발생하는 예외(Exception) 상황(예: 존재하지 않는 인덱스 접근, 객체가 null인 경우)으로 인해 프로그램이 갑자기 종료되는 것을 방지하고, 안전하게 오류를 처리하여 다음 작업을 이어서 진행할 수 있게 하는 것이 목표입니다.
- **구문:** 자바스크립트는 Java와 유사하게 `try`, `catch`, `finally` 블록을 사용합니다.
    - `try`: 예외가 발생할 가능성이 있는 코드를 묶습니다.
    - `catch`: 예외가 발생했을 때 실행할 처리 로직을 작성합니다.
    - `finally`: 예외 발생 여부와 관계없이 **항상** 실행되어야 하는 정리 코드(예: 파일 닫기)를 작성합니다.
- **특징 (자바스크립트):**
    - 자바와 달리 자바스크립트의 `catch` 블록은 여러 타입을 받아낼 수 없습니다. (Weakly-typed 언어의 특성)
    - 개념적으로는 자바와 유사하지만, 동작 원리와 문법적 특징에 차이가 있습니다.

## 4. 동기/비동기 처리와 자바스크립트 실행 메커니즘
- **동기 (Synchronous) vs. 비동기 (Asynchronous):**
    - **동기:** 코드가 위에서 아래로 순차적으로 실행되며, 한 작업이 끝날 때까지 다음 작업은 기다립니다.
    - **비동기:** 시간이 오래 걸리거나 (I/O, 서버 통신, 타이머 등) 자바스크립트 단일 스레드가 처리하기 어려운 작업을 백그라운드에서 처리하고, 완료되면 나중에 결과를 받아와 처리하는 방식입니다.
- **자바스크립트의 특징:**
    - 자바스크립트는 **단일 스레드** 환경에서 동작합니다.
    - 웹 브라우저에서 실행되며, 웹 API(Web API)와 같은 브라우저의 기능(DOM 조작, `setTimeout` 등)은 자바스크립트가 직접 처리할 수 없는 부분입니다.
    - **Web API의 역할:** 자바스크립트가 처리할 수 없는 비동기 작업은 Web API에 위임합니다.
- **실행 메커니즘 (매우 중요):**
    1. **Web API 분리:** 비동기 작업이 발생하면 (예: `setTimeout(fn, 2000)`), 해당 작업은 자바스크립트 스택(Call Stack)에서 분리되어 Web API 영역으로 넘어갑니다.
    2. **콜백 큐 대기:** Web API가 작업을 처리한 후, 완료되어 호출해야 할 함수(콜백)는 **콜백 큐(Callback Queue)**라는 곳에 대기합니다.
    3. **이벤트 루프 (Event Loop):** 이벤트 루프가 지속적으로 작동하며, 자바스크립트 스택이 완전히 비워지는(즉, 현재 실행할 코드가 없는) 순간을 감지합니다.
    4. **함수 실행:** 스택이 비었을 때, 콜백 큐에 대기 중인 콜백 함수를 다시 자바스크립트 스택으로 "끌어 올려(Push)" 실행합니다.

*핵심 결론: 비동기 코드는 즉시 실행되는 것이 아니라, 메인 스레드가 여유가 될 때(스택이 비었을 때) 순차적으로 실행된다.*

## 1. 자바스크립트 비동기 처리의 이해
*   **기본 원리 (단일 스레드)**: 자바스크립트는 기본적으로 단일 스레드(Single Thread)로 동작하며, 모든 작업은 순차적으로 실행됩니다.
*   **비동기 처리의 필요성**: 네트워크 통신(AJAX), 시간 지연 등 시간이 오래 걸리는 작업(I/O 작업)을 동기적으로 처리할 경우, 자바스크립트 전체가 멈추는 현상(Blocking)이 발생하여 사용자 경험을 저해합니다.
*   **비동기 작동 방식**: 따라서 자바스크립트 엔진은 시간이 걸리는 작업을 감지하면, 해당 작업을 즉시 처리하는 대신 비동기 방식으로 '던져놓고(Fire and Forget)' 다음 코드를 계속 실행합니다.
*   **핵심 개념**: `setTimeout`, `setInterval`과 같은 함수는 대표적인 비동기 함수이며, 이들은 콜백 메커니즘을 통해 나중에 결과를 처리합니다.

## 2. 콜백 함수(Callback Function)와 그 한계
*   **개념**: 콜백 함수는 **비동기 작업이 완료된 후에 실행되도록** 전달받는 함수입니다.
*   **작동 원리**: 시간이 걸리는 작업을 호출할 때, "이 작업이 끝나면 이 함수(콜백)를 실행해 줘"라고 같이 전달해야 합니다.
*   **구조적 문제 (콜백 지옥, Callback Hell)**: 여러 개의 비동기 작업을 순차적으로 처리해야 할 경우, 콜백 함수를 계속 중첩해서 사용해야 합니다. 이는 코드를 매우 복잡하게 만들고(Nested Function), 가독성이 떨어지며, 에러 처리(Error Handling)가 어렵다는 치명적인 단점(콜백 지옥)을 가집니다.

## 3. Promise 객체: 비동기 로직의 체계화
*   **등장 배경**: 콜백 지옥을 해결하고 비동기 로직을 더 구조화하고 직관적으로 만들기 위해 등장했습니다.
*   **개념**: Promise는 **"미래에 값이 존재할 것을 약속하는 객체"**입니다. 비동기 작업의 최종 성공 또는 실패 결과를 관리하는 래퍼(Wrapper) 역할을 합니다.
*   **주요 상태 및 메서드**:
    *   **`resolve` (성공)**: 비동기 작업이 성공적으로 완료되었을 때 호출됩니다.
    *   **`reject` (실패)**: 비동기 작업 중 오류(에러)가 발생했을 때 호출됩니다.
    *   **`.then()`**: Promise가 성공했을 때(resolve) 실행될 로직을 등록합니다.
    *   **`.catch()`**: Promise가 실패했을 때(reject) 실행될 예외 처리 로직을 등록합니다.
*   **장점**: 콜백 함수를 계속 중첩할 필요 없이, `.then()`을 연결하는 체이닝(Chaining) 방식으로 코드를 순차적이고 깔끔하게 구성할 수 있습니다.

## 4. Async/Await: 비동기 코드의 간결한 구현
*   **목적**: Promise를 사용한 `.then()` 체이닝 방식조차도 여전히 복잡하게 느껴질 수 있기 때문에, 비동기 코드를 **동기 코드처럼 보이게(Synchronous)** 작성하기 위해 도입된 문법적 설탕(Syntactic Sugar)입니다.
*   **핵심 문법**:
    *   **`async`**: 함수 선언 앞에 붙여 이 함수가 비동기 함수임을 명시합니다. (이 함수는 Promise를 반환합니다.)
    *   **`await`**: `await` 키워드를 사용하면, 해당 값이 Promise 객체일 때, 값이 실제로 반환될 때까지 코드의 실행을 "일단 멈추고(Wait)" 기다립니다.
*   **작동 방식**: `await` 덕분에 개발자는 비동기 로직을 마치 동기적으로 순차 진행되는 것처럼 코드를 작성할 수 있습니다.
*   **최대 장점**: 콜백으로 감싸거나 `.then()`으로 체이닝할 필요 없이, 순차적이고 가독성이 높은 코드로 비동기 작업을 처리할 수 있게 됩니다.

## 5. 요약 및 다음 학습 방향
*   **핵심 정리 (기술 발전 과정)**:
    1.  **비동기 문제 인식**: 시간이 걸리는 작업을 순차적으로 처리해야 하는 어려움 발생.
    2.  **콜백 함수**: 문제 해결을 위한 최초의 방식. (단점: 콜백 지옥, 가독성 및 에러 처리 어려움).
    3.  **Promise**: 비동기 결과를 약속하는 객체. 체이닝을 통해 구조화 개선. (개선된 순차성 확보).
    4.  **Async/Await**: Promise를 기반으로, 비동기 코드를 마치 동기 코드처럼 작성할 수 있도록 문법적 편의를 제공. (가장 간결함).
*   **결론**: 비동기 처리가 매우 어려운 영역이므로, 이 개념들을 완벽히 이해하는 것은 시간이 필요합니다. 핵심은 **"시간이 걸리는 비동기 작업을 어떻게 하면 순차적이고 오류 없이 관리할 수 있을까"** 하는 흐름을 이해하는 것입니다.
*   **다음 학습 주제**:
    *   **DOM 조작**: 브라우저의 요소를 가져오고 이벤트(Event)를 걸어 조작하는 것이 자바스크립트의 핵심 활용 영역입니다.
    *   **Git**: 협업 및 프로젝트 관리를 위해 버전 관리 시스템(Git) 사용법을 숙지하는 것이 중요합니다.



1. 자바스크립트는 기본적으로 단일 스레드로 동작한다.
한 번에 하나의 코드만 실행한다.

setTimeout, 이벤트 처리, fetch 같은 비동기 작업은 브라우저 또는 런타임 환경에 맡겨두고 자바스크립트는 그 작업이 끝날 때까지 기다리지 않고 다음 코드를 먼저 실행한다.

작업이 끝나면 등록해둔 callback 함수가 실행 대기열에 들어가고,
콜 스택이 비어 있을 때 이벤트 루프에 의해 실행된다.

2. callback 함수 : 작업 끝나면 이 함수 실행해줘 하고 미리 맡겨두는 방식
작업이 중첩되면 코드가 가독성이 떨어진다 -> 콜백지옥

3. Promise는 비동기 작업의 결과를 나중에 받을 수 있게 해주는 객체다.
Promise는 세 가지 상태를 가진다.

pending: 작업이 아직 끝나지 않은 상태
fulfilled: 작업이 성공적으로 끝난 상태
rejected: 작업이 실패한 상태

비동기 작업이 성공하면 resolve가 호출되고,
실패하면 reject가 호출된다.

then()을 사용하면 성공 결과를 이어서 처리할 수 있고,
catch()를 사용하면 실패를 처리할 수 있다.

Promise를 사용하면 콜백 중첩을 줄이고,
then 체이닝을 통해 비동기 흐름을 더 읽기 쉽게 만들 수 있다.

4. async await : async 함수 안에서 await 를 사용해 비동기 작업이 끝날 때까지 기다렸다가 다음 줄로 넘어감
비동기 코드를 동기 코드처럼 읽히게 만들기 위해, 가독성 좋아짐, 에러 처리 편함, 디버깅 쉬워짐.
await은 Promise가 처리될 때까지 async 함수의 실행을 잠시 멈춘다.
Promise가 fulfilled 상태가 되면 resolve된 값을 반환한다.
그래서 그 값을 변수에 담을 수 있다.
